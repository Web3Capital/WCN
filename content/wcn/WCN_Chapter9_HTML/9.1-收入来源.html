Chapter 09 · 9.1

WCN 的收入不应只来自未来，而必须在当前阶段就有现实来源。

一个真正可持续的系统，必须在 Token 之前就建立现金收入。WCN 的收入来源不是单点收费，而是围绕节点、Deal、服务和 Agent 形成多层收入结构。

Section 1

收入地图

WCN 的收入可以分为四类：节点席位费、服务费、成交 / 成功费、Agent 相关收入。

这四类收入共同构成早期现金流，而不是依赖单一来源。

WCN 的商业模型是“网络收入 + 服务收入 + 成交收入 + Agent 收入”的组合。
节点席位费
节点进入网络、获得位置和权限的基础收费。
服务费
围绕法务、安全、增长、研究、开发等服务产生的收入。
成交 / 成功费
当 deal 或闭环成立后，按照规则收取的商业费用。
Agent 收入
由 Agent 配置、运行和执行支持带来的收入。
Section 2

为什么收入必须多元

如果只依赖一种收入，系统会非常脆弱。

多元收入结构可以让 WCN 在不同阶段保持更高韧性。

Section 3

收入和节点网络的关系

收入不是附加层，而是网络本身运作的结果。

节点越真实、闭环越多、服务越成熟，收入结构就越稳。
