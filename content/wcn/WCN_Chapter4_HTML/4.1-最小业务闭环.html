4.1 · 最小业务闭环

WCN 的最小单位，不是内容、关系或活跃度，而是一个可验证的业务闭环。

只有当资源进入网络、任务被推进、结果形成、证据被提交并进入 PoB，WCN 才真正开始运作。这个最小闭环决定了系统的所有后续结构。

最小闭环 6 步

资源进入项目、资本、服务或分发被带入系统。
机会结构化机会进入 Deal / Task。
执行推进人和 Agent 协同推进。
结果形成融资、服务、增长、合作等结果出现。
证据提交结果进入 Proof Desk。
PoB 记账通过审核后进入结算输入。
Definition

什么叫“最小业务闭环”？

最小业务闭环，是指一个最小但完整的业务结果链路。

它至少包含三件事：

有资源输入必须有项目、资本、服务、分发、地区资源或 Agent 任务进入系统。
有执行过程必须有人或 Agent 实际推动了协作，而不是停留在口头连接。
有结果证明必须留下可验证结果和对应证据。
有结算资格只有通过 PoB 审核，结果才进入长期价值层。
Flow

最小闭环的标准路径

节点引入资源 → 机会进入 Deal / Task → 人与 Agent 执行 → 结果形成 → 证据进入 Proof Desk → 生成 PoB

这一条路径说明，WCN 不是为了记录所有动作，而是为了把“真正推动结果成立的动作”从大量噪音里区分出来。

Principle

为什么最小闭环这么重要？

它决定什么能被记录
没有闭环，动作只是动作，不进入长期价值系统。
它决定什么能被奖励
不是所有参与都值得结算，只有结果成立后才有意义。
它决定网络质量
网络不是由人数定义，而是由真实闭环密度定义。
它决定 PoB 的可信度
PoB 不是主观打分，而是建立在闭环之上的证明机制。
