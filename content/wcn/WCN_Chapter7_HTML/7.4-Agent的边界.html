7.4 · 边界

AI Agent 的价值越大，边界就越重要。

WCN 不是让 Agent 自由发挥，而是让它在清晰边界内最大化发挥。越靠近资金、法律、合同、治理和声誉，越需要把边界写死。

本页作用
定义 Agent 权限边界
核心主题
可做与不可做必须被写清
阅读重点
读写边界、审批边界、责任边界

核心原则

低风险动作可自动化研究、摘要、提醒、整理等。
高风险动作必须有人审批合同、付款、法律结论、外部承诺等。
越靠近结算与治理，边界越严风险越高，越不能默认交给 Agent。
Can Do

Agent 可以做什么？

研究与整理
摘要、对比、尽调初筛、材料结构化。
流程与纪要
会议纪要、待办拆解、催办和状态更新。
增长与监控
内容辅助、归因分析、指标监控、异常预警。
任务执行支持
在明确权限下完成低风险、重复性工作。
Cannot Do

哪些事不能直接交给 Agent？

不能替代签约Agent 不能最终代表任何主体完成合同签署。
不能替代付款Agent 不能单独触发高风险资金动作。
不能给法律 / 税务结论高风险专业结论必须由人类专业方给出。
不能最终批准 PoB审核和结论必须保留给人类审核层。
不能越权对外承诺尤其不能做收益、治理、监管相关承诺。
不能绕过日志和人工接管任何关键动作都必须可回溯、可制止。
Why Boundaries Matter

为什么要把边界写得这么清楚？

没有边界，AI 不是“更强执行层”，而会变成“更大风险源”。

WCN 需要的是受控的、可信的、可进入结算的 Agent，而不是不可解释、不可追责的自动化噪音。
