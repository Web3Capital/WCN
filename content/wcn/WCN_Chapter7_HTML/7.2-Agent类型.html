7.2 · Agent 类型

不是所有 Agent 都一样。不同类型 Agent，必须承担不同类型的任务。

WCN 需要的不是一个万能 Agent，而是一组在任务上清晰分工的 Agent 系统。只有这样，日志、权限、审核和结算才有可能被准确管理。

本页作用
定义 Agent 分类
核心主题
不同 Agent = 不同任务身份
阅读重点
Research、Deal、Growth、Execution、Liquidity

主要类型

Research Agent研究、摘要、初筛、整理。
Deal Agent匹配、引荐、状态推进。
Growth Agent内容、触达、归因、增长动作。
Execution Agent待办、回执、催办、流程执行。
Liquidity Agent流动性监控、预警、策略支持。
Types

五种典型 Agent

Research Agent
负责信息收集、摘要、初筛、对比、尽调辅助与结构化输出。
Deal Agent
负责项目与资本匹配、引荐准备、状态推进与 deal 推荐。
Growth Agent
负责内容生成、传播辅助、增长归因与渠道支持。
Execution Agent
负责纪要、待办、提醒、回执收集、节点协作节奏管理。
Liquidity Agent
负责流动性监控、上所准备、市场状态与基础策略支持。
Difference

它们最本质的区别是什么？

Research Agent 面向信息核心是“理解和整理”。
Deal Agent 面向匹配核心是“连接与推进”。
Growth Agent 面向传播核心是“内容与增长动作”。
Execution Agent 面向流程核心是“确保任务被完成”。
Liquidity Agent 面向市场核心是“交易与流动性状态”。
Why Classification Matters

为什么一定要把 Agent 分类？

因为不分类，权限会混乱，日志会混乱，责任会混乱，价值也无法被准确结算。

分类清楚后，系统才能知道：

谁能看什么不同 Agent 的可见数据不同。
谁能做什么不同 Agent 的执行动作不同。
谁该留下什么日志不同任务类型对应不同留痕标准。
谁如何进入 PoB只有被采纳并进入闭环的 Agent 输出才有价值。
