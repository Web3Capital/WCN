7.3 · Agent 生命周期

Agent 不是随便接进来就一直运行，它也必须有生命周期。

WCN 里的 Agent 不是“永久外挂”。它们必须像节点一样，经历注册、授权、运行、复核、升级、暂停和退出，才能形成可信执行层。

本页作用
定义 Agent 状态机
核心主题
Agent 也必须被治理
阅读重点
注册、权限、运行、暂停、淘汰

典型生命周期

Registered已登记，但尚未授权。
Authorized具备正式任务权限。
Active正在执行任务。
Paused / Suspended因问题被暂停。
Upgraded / Retired升级或退出。
States

Agent 的关键状态

Registered
Agent 已被登记，但还没进入正式业务流程。
Authorized
Agent 被赋予明确任务、数据和权限边界。
Active
Agent 正在执行任务，并持续写入日志。
Paused / Suspended
因质量、越权或风险问题暂停。
Upgraded
Agent 升级模型、能力或权限等级。
Retired
Agent 因不再适用或不达标而退出系统。
Flow

生命周期路径

注册 → 授权 → 上线 → 日志记录 → 审核 / 复核 → 升级 / 暂停 / 淘汰

Agent 生命周期的设计目标，不是增加复杂度，而是保证：

每个 Agent 都有身份知道是谁、做什么、属于谁。
每个 Agent 都有边界不能无约束读取和执行。
每个 Agent 都有日志可以回溯与审查。
每个 Agent 都能被淘汰质量差或越权的 Agent 不能无限存在。
Why It Matters

为什么这一页重要？

因为如果 Agent 没有生命周期，系统会出现两个问题：

Agent 失控
没有授权与暂停机制，Agent 容易越权或持续输出低质量结果。
Agent 无法进入 Proof
没有身份和日志，Agent 的贡献无法被验证和结算。
