## 这一章回答的，不只是“你是谁”，更是“你为什么成立”。

WCN 不是单一句子就能讲完的项目。它既不是传统意义上的公链，也不是单一的社群、基金或工具， 而是一个试图把节点、执行、证据与结算组织进同一系统的 Web3 + AI 业务网络。

## 第一章的任务，是为后面所有章节建立统一坐标系。

如果第一章没有建立清楚，后面的节点、Agent、PoB、商业模式和治理内容都会被误读。 因此，这一章不是背景介绍，而是整个文档站的基础坐标。

### 从定义开始，而不是从功能开始。

很多项目一开始就讲功能、讲机制、讲代币，但没有先回答“自己到底是什么”。 WCN 的第一章反过来做：先定义身份与边界，再展开系统逻辑。这样读者才能知道后续章节的解释框架。

- **如果你是投资人**：这一章帮你判断：WCN 是概念拼装，还是有明确系统定位的项目。
- **如果你是节点或合作方**：这一章帮你判断：为什么 WCN 值得参与，以及它不是哪类短期项目。
- **如果你是内部团队成员**：这一章帮你统一外部口径，避免后续叙事分叉与表达漂移。
