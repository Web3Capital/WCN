## 第一段：资源进入系统

系统运作的起点，不是表单，不是页面，而是资源被带进网络。

资源进入后，系统的下一步不是立刻匹配，而是先进入 Deal 或 Task 结构。

## 第二段：Deal 与 Task 被推进

一旦资源进入系统，它们必须从“关系”变成“工作流”。

<Callout type="warning">
机会进入 Deal Room，执行进入 Task Dispatch，人和 Agent 在明确责任下开始推进。
</Callout>

## 第三段：结果进入 Proof Desk 与 PoB

一旦结果形成，系统必须从执行逻辑切换到证明逻辑。

<Callout type="warning">
WCN 的关键不在于“结果做成了”，而在于“结果是否被系统性证明并进入长期价值层”。
</Callout>
