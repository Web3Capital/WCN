## 四类席位（可叠加）

<CardGrid cols={2}>
  <Card title="功能席位" icon="⚙️">资本、项目、服务、分发、区域、Agent 等主责身份。决定默认工作流模板与 KPI 类别（如资本侧为响应时效 + 投资转化率，服务侧为交付准时率 + 复检通过率）。</Card>
  <Card title="地区席位" icon="🌍">国家/城市/时区覆盖；常伴**排他或限额**（例如同一辖区仅 N 个主节点），避免内部抢单与声誉稀释。</Card>
  <Card title="行业席位" icon="🏷️">AI、RWA、支付、基础设施等标签；驱动 L1–L2 匹配权重与联合品牌活动资格。</Card>
  <Card title="任务席位" icon="🎯">绑定具体 Deal ID、Task ID 或活动代号；权限**时间盒**化，任务结束即回收或降级——类似项目制顾问门禁。</Card>
</CardGrid>

---

## 四类授权（与责任对称）

<CardGrid cols={2}>
  <Card title="访问权（Read）" icon="👁️">可见的 Deal Room、文档版本、指标看板范围。可按「需知原则」细分至字段级（对标投行数据室的 watermarked PDF 策略）。</Card>
  <Card title="参与权（Engage）" icon="🚪">可投标、评论、认领 Task、加入会议序列。通常要求 Conflict 声明前置。</Card>
  <Card title="操作权（Act）" icon="🔧">链上或 API 级动作：铸造凭证、触发分配、调用 Agent 批处理。多需**双人复核**或时间锁。</Card>
  <Card title="结算资格（Settle）" icon="💰">进入 PoB 归因与价值分配路径的条件：证据包齐全、Reviewer 签字、无开放争议工单。未满足则**权限可冻结**而身份仍存在。</Card>
</CardGrid>

<Callout type="info">**席位**回答「你代表哪类机构责任」；**授权**回答「本轮你能动哪些对象」。升级席位不自动等于扩张授权——须通过履约与安全审查。</Callout>

---

## 系统为什么依赖这一层

没有席位与授权，协作图谱无法自动化——匹配算法、风控规则、审计日志都失去主语。WCN 从「谁认识谁」升级为「**谁在何种授权下于何时对何对象做了什么**」——与交易所订单日志、终端用户审计追踪同一粒度需求。

<Statement>席位 + 授权 + 证据链，是机构级网络可规模化的最小架构；缺一则退回人情与口头承诺。</Statement>
