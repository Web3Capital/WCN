## 与三类常见身份技术的对照

| 方案 | 强项 | 局限 | Node NFT 的取舍 |
| --- | --- | --- | --- |
| **ENS** | 人类可读、广泛集成 | 不原生表达复杂权限与组织关系 | 可与 ENS 并列：ENS 指人/品牌，Node NFT 指**机构席位** |
| **SBT（灵魂绑定代币）** | 强调不可转让、声誉绑定 | 升级、席位迁移、合规冻结场景不灵活 | WCN 可采用**可转移性由政策决定**的凭证（可锁仓、可收回映射） |
| **W3C Verifiable Credentials** | 标准隐私模型、选择性披露 | 链上可组合性与钱包原生展示较弱 | 关键声明可双写：VC 用于合规披露，NFT 用于**协议内快速门禁** |

<CardGrid cols={2}>
  <Card title="身份载体" icon="id-card">链上证明「某地址被绑定到某节点主体与席位代际」；刷新权限时常通过 metadata 版本或侧链状态根，而非频繁换钱包。</Card>
  <Card title="权限载体">作为调用 API、进入合约函数或解锁前端路由的**必要非充分**条件；最终仍以服务端 ACL 与合规 flag 为准——类似硬件 Token + 服务器 session。</Card>
  <Card title="记录载体（演进）">可挂载信誉分、等级、历史 Task 哈希指针；完整证据仍多在链下对象存储 + Proof Desk，避免主网 gas 与隐私问题。</Card>
  <Card title="可编程性">支持与激励合约、联合营销白名单、跨合作伙伴 SDK 对接；须在智能合约层限制**不可将 Node NFT 表述为证券或存款凭证**。</Card>
</CardGrid>

## Node NFT 明确不是什么

<CardGrid cols={2}>
  <Card title="不是理财产品">不得承诺本金、固定收益或与管理规模挂钩的自动分润——与持牌基金/存款的法律边界清晰区隔。</Card>
  <Card title="不是纯头像">视觉层可有可无；价值来自与 WCN 账户、协议义务、KPI 的绑定关系。</Card>
  <Card title="不是无许可铸造">铸造权在网络治理/运营侧；开放 mint 会破坏席位稀缺性与风控前提。</Card>
  <Card title="不是链上万能钥匙">丢失私钥、主体解约、合规触发时，应能通过**冻结映射、轮换地址、作废代际**等运维手段处置。</Card>
</CardGrid>

## 实施原则

<Callout type="warning">**业务网络先行**：先有真实 Deal、Task、Reviewer 与证据标准，再扩大 NFT 承载范围。否则与「先卖会员卡再装修会所」同类风险——声誉与监管双面承压。</Callout>

<Statement>当前阶段应把 Node NFT 理解为**专业网络的数字化席卡**：链上可验证、链下可问责、系统可一键收回——而非链上投机标的。</Statement>
