# Node NFT 的核心不是投机，而是把节点位置、身份与权限逐步数字化。

<MetaGrid>

  <Meta label="当前阶段" value="不是收益产品，是数字化接口" />
  <Meta label="前提条件" value="先有节点和闭环，再有 NFT 承载" />

## Node NFT 的三重角色

<CardGrid cols={2}>
  <Card title="身份载体" icon="🪪">证明某个节点在系统中的正式位置。</Card>
  <Card title="权限载体" icon="🔑">映射访问权、参与权和部分未来系统能力。</Card>
  <Card title="记录载体" icon="📊">未来可映射节点等级、信誉与历史。</Card>
  <Card title="数字化位置" icon="⛓️">让节点席位逐步转向可编程承载。</Card>
</CardGrid>

---

## Node NFT 不是什么

<CardGrid cols={2}>
  <Card title="不是投资产品" icon="🚫">不能被写成保底收益或自动回报逻辑。</Card>
  <Card title="不是纯头像凭证" icon="🚫">必须和正式位置、权限、责任绑定。</Card>
  <Card title="不是空壳收藏品" icon="🚫">意义来自系统关系，而不是图像本身。</Card>
  <Card title="不是金融化产品" icon="🚫">当前阶段不应被表达为收益承诺。</Card>
</CardGrid>

---

## 为什么 WCN 需要 NFT 形态

当网络成熟后，位置、权限、等级和长期记录都需要更标准化、更数字化的表达。NFT 是一种可能的承载方式，但前提是业务网络先成立。

<Callout type="warning">先有节点和闭环，再有 NFT 承载——不是先卖 NFT 再找意义。</Callout>

<Statement>当前阶段的 Node NFT 是身份、权限、位置与未来长期数字化接口，而非收益承诺。</Statement>
