# WCN 的最小单位，不是内容、关系或活跃度，而是一个可验证的业务闭环。
</Hero>

<MetaGrid>
  <Meta label="本页作用" value="定义 WCN 最小运作单元" />
  <Meta label="核心主题" value="闭环而不是表面动作" />
  <Meta label="阅读重点" value="资源、执行、证明、PoB" />
</MetaGrid>

<Statement>最小业务闭环，是指一个最小但完整的业务结果链路。</Statement>

## 闭环包含什么

<Steps>
  <Step title="有资源进入">节点把项目、资本、服务或地区资源引入网络，形成可被推进的机会。</Step>
  <Step title="有执行推进">人和 Agent 在 Deal / Task 中把机会推进成可交付的结果。</Step>
  <Step title="有结果被证明">证据被提交、审核通过、归因明确，结果进入 PoB 记录。</Step>
</Steps>

---

## 标准路径

<Callout type="info">
节点引入资源 → 机会进入 Deal / Task → 人与 Agent 执行 → 结果形成 → 证据进入 Proof Desk → 生成 PoB
</Callout>

这一条路径说明，WCN 不是为了记录所有动作，而是为了把真正推动结果成立的动作从大量噪音里区分出来。

---

## 为什么最小闭环如此重要

<CardGrid cols={2}>
  <Card title="定义系统边界" icon="📐">闭环是 WCN 区分噪音和价值的唯一标准。没有闭环，系统就无法判断什么值得记录。</Card>
  <Card title="驱动激励设计" icon="💡">所有激励、归因和结算都围绕闭环展开。没有闭环就没有结算入口。</Card>
  <Card title="锚定执行质量" icon="✅">闭环要求每一步都有 owner、有状态、有输出，确保执行不会停在空转层面。</Card>
  <Card title="形成网络记忆" icon="💎">每一个闭环都会沉淀为系统资产——信誉、记录、归因、长期位置。</Card>
</CardGrid>

<Callout type="tip">
WCN 的一切后续结构——激励、结算、信誉、治理——都以最小业务闭环为原点。没有闭环，系统就没有起点。
</Callout>
