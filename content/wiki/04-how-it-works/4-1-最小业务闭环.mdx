# WCN 的最小单位不是内容、关系或活跃度，而是一个可验证的业务闭环。
</Hero>

<MetaGrid>
  <Meta label="核心定义" value="有资源进入 + 有执行推进 + 有结果被验证 = 闭环" />
  <Meta label="关键意义" value="闭环是系统区分噪音和价值的唯一标准" />
  <Meta label="TradFi 对标" value="PE/VC 的 Deal Completion Rate" />
</MetaGrid>

<Statement>最小业务闭环 = 资源被引入 + 任务被执行 + 结果被验证。三者缺一，都不算闭环。</Statement>

---

## 闭环的三个必要条件

<Steps>
  <Step title="有资源进入">节点把真实的项目、资本、服务或区域资源引入网络。资源必须是具体的——一个在融资的项目、一笔可配置的资金、一项可交付的服务——而不是「认识一些人」。</Step>
  <Step title="有执行推进">人和 Agent 在 Deal / Task 中把资源从「机会」推进到「结果」。推进意味着有人负责、有动作发生、有状态变化——而不是「拉了个群聊了聊」。</Step>
  <Step title="有结果被验证">证据被提交、审核通过、归因明确。结果进入 PoB 记录。验证意味着有第三方可查的证据——而不是「他说做成了」。</Step>
</Steps>

---

## 什么算闭环，什么不算

这是 WCN 与所有「活跃度奖励」系统的根本区别：

<CardGrid cols={2}>
  <Card title="算闭环">种子轮融资完成（有协议 + 打款记录）；法务审计交付通过验收（有 SOW + 交付件）；做市服务上线（有合同 + 链上流动性数据）；项目被推荐并成功上所（有沟通记录 + 上线证据）。</Card>
  <Card title="⊘ 不算闭环" icon="ban">介绍了两个人认识但没有后续；分享了 Deck 但没有推进；开了会但没有行动项；Agent 做了研究但没人采纳；发了推文但没有业务转化。</Card>
</CardGrid>

> **行业类比**：在 TradFi PE/VC 中，「Deal Completion Rate」是核心指标——看的不是你接触了多少项目，而是你真正 close 了几个。一个 VC 可能一年看 3000 个项目，但 Deal Completion 只有 10-15 个。WCN 的闭环概念等价于此：只有 close 了的 Deal 才进入价值层。

<Callout type="warning">
判断标准非常简单：如果拿掉这个动作，业务结果还是会发生，那这个动作就不构成闭环的一部分。只有对结果有因果贡献的参与，才算。
</Callout>

---

## 标准闭环路径

<Callout type="info">
节点引入资源 → 机会进入 Deal / Task → 人与 Agent 执行推进 → 结果形成 → 证据进入 Proof Desk → 审核通过 → 生成 PoB 记录 → 进入结算层
</Callout>

关键特征：路径上的每一步都有明确的 **owner**（谁负责）、**status**（当前状态）和 **output**（产出了什么）。没有「无 owner 的任务」和「无状态的进展」。

---

## 为什么 Web3 至今没有这个概念

<CardGrid cols={2}>
  <Card title="没有统一的结果定义">Web3 的「成功」定义高度碎片化——有人看 TVL，有人看社区人数，有人看 Token 价格。没有一个系统把「真实业务完成」作为统一度量。</Card>
  <Card title="没有证据基础设施">TradFi 有完整的审计链条（四大、律所、托管行）。Web3 的多方协作中，证据散落在 Telegram、Email、Google Docs 和区块链上，没有统一收集和审核的系统。</Card>
  <Card title="激励设计奖励过程而非结果">空投、积分、签到、社交任务——这些机制奖励的是「做了动作」，而不是「产生了结果」。系统因此被噪音淹没。</Card>
  <Card title="归因困难导致回避">多方协作中，谁对结果贡献了多少，极难判断。大多数项目选择回避这个问题（平分或靠关系谈判），而不是建立系统性的归因机制。</Card>
</CardGrid>

---

## 闭环为什么是系统的原点

<CardGrid cols={2}>
  <Card title="定义价值边界">闭环是 WCN 区分「值得记录」和「不值得记录」的唯一标准。没有闭环 → 没有 PoB → 没有结算入口 → 没有价值分配。</Card>
  <Card title="驱动激励设计">所有激励和结算都围绕闭环展开。这意味着系统的资源分配天然指向「能推动闭环的人」，而非「能制造噪音的人」。</Card>
  <Card title="锚定执行质量">闭环要求每一步都有 owner、有状态、有输出。这迫使参与者认真执行，而非假装忙碌。</Card>
  <Card title="积累网络价值">每一个验证过的闭环都沉淀为系统资产：节点信誉、匹配精度、归因数据、Agent 训练素材。时间越长，系统越难被替代。</Card>
</CardGrid>

<Callout type="tip">
WCN 的一切后续结构——PoB、结算、信誉、治理——都以最小业务闭环为原点。理解闭环，就理解了 WCN 为什么能从噪音中提取真实价值。
</Callout>
