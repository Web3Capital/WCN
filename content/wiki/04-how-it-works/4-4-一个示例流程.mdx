## 场景设定

**项目方**：一个 AI + DeFi 基础设施项目（团队在新加坡，技术在欧洲），正在进行 $2M 种子轮融资。需要：

<CardGrid cols={2}>
  <Card title="找到合适的资本方" icon="💰">偏好：了解 AI + DeFi 交叉领域的基金，能接受 SAFE + Token Warrant 结构，票面 $200K-$500K。</Card>
  <Card title="完成法务结构" icon="📑">需要：SAFE 协议审核、Token Warrant 条款、新加坡实体合规检查、投资人 KYC 流程。</Card>
</CardGrid>

**WCN 网络中的参与方**：

<CardGrid cols={3}>
  <Card title="资本节点 A" icon="🏦">香港基金，偏好 AI + Infra 赛道，种子轮票面 $300K-$500K。</Card>
  <Card title="资本节点 B" icon="🏦">迪拜家族办公室，偏好 DeFi 基础设施，票面 $500K-$1M。</Card>
  <Card title="服务节点 C" icon="⚖️">新加坡法务团队，擅长 SAFE/SAFT + Token Warrant 结构。</Card>
</CardGrid>

<CardGrid cols={2}>
  <Card title="Research Agent" icon="🔬">负责项目资料结构化、竞品分析、风险标记、投资备忘录草稿。</Card>
  <Card title="Execution Agent" icon="⚡">负责会议纪要、待办追踪、催办提醒、材料收集状态。</Card>
</CardGrid>

## 14 天完整流程

### Day 1-2：资源录入与匹配

<CardGrid cols={1}>
  <Card title="Day 1：项目录入" icon="📥">项目节点提交资料至 Project Intake：Deck、Tokenomics 草案、团队背景、技术文档、融资条款（$2M SAFE + Token Warrant）。Research Agent 在 2 小时内生成结构化摘要 + 风险清单 + 竞品对比。</Card>
  <Card title="Day 2：匹配与 Deal Room 创建" icon="🤝">Deal Agent 基于资本节点偏好匹配出 A 和 B。系统创建 Deal Room，Deal Owner（由引荐的区域节点担任）被指定，服务节点 C 被加入法务 Track。所有参与方看到同一份结构化资料。</Card>
</CardGrid>

### Day 3-7：执行推进

<CardGrid cols={1}>
  <Card title="Day 3：首次会议" icon="📞">Deal Owner 组织项目方与资本节点 A 的视频会议。Execution Agent 实时生成纪要，自动提取 5 项 Action Items 并分配 owner。会后 30 分钟内，所有参与方收到结构化纪要。</Card>
  <Card title="Day 4-5：尽调与法务并行" icon="⚡">资本节点 A 要求补充技术审计报告和创始人背景调查。Research Agent 2 小时内整理完毕。同时，服务节点 C 开始审核 SAFE + Token Warrant 条款，在 Day 5 出具初步法律意见。</Card>
  <Card title="Day 6：第二轮会议 + 资本节点 B 进入" icon="📞">资本节点 A 确认跟进意向（$400K）。资本节点 B 看完资料后要求单独会议。Deal Room 同时管理两条资本线，状态分别追踪。</Card>
  <Card title="Day 7：条款谈判" icon="📋">项目方和资本节点 A 就估值和 Token Warrant 比例达成一致。服务节点 C 出具最终法律意见。Execution Agent 追踪所有待签文件的状态。</Card>
</CardGrid>

### Day 8-11：协议签署与打款

<CardGrid cols={1}>
  <Card title="Day 8-9：协议签署" icon="✍️">资本节点 A 签署 SAFE ($400K) + Token Warrant。资本节点 B 签署 SAFE ($500K) + Token Warrant。服务节点 C 完成所有协议的合规审核。</Card>
  <Card title="Day 10-11：打款确认" icon="💸">资本节点 A 完成 $400K 打款（银行 Wire）。资本节点 B 完成 $500K 打款（$300K Wire + $200K USDC）。Execution Agent 追踪打款状态，自动关联银行确认和链上记录。</Card>
</CardGrid>

### Day 12-14：证据收集与 PoB

<CardGrid cols={1}>
  <Card title="Day 12：证据打包" icon="📎">Deal Owner 将以下证据提交至 Proof Desk：已签署的 SAFE 协议（2 份）、Token Warrant（2 份）、打款银行确认（2 份）、链上 USDC 转账哈希（1 笔）、法律意见书（1 份）、所有会议纪要（4 份）。</Card>
  <Card title="Day 13：审核" icon="🔍">独立审核人检查证据完整性和真实性。确认：融资金额 $900K（本轮首关）、协议条款与录入一致、法务交付完整、时间线合理。</Card>
  <Card title="Day 14：PoB 生成 + 归因" icon="✅">审核通过，系统生成 PoB Event。归因分配如下（见下方）。</Card>
</CardGrid>

## 归因分配明细

<CardGrid cols={3}>
  <Card title="Deal Owner（区域节点）" icon="👑">归因：30%。理由：引荐项目方、创建 Deal Room、组织全部会议、推进签约进度。</Card>
  <Card title="资本节点 A" icon="🏦">归因：20%。理由：完成尽调、签约、打款 $400K。</Card>
  <Card title="资本节点 B" icon="🏦">归因：20%。理由：完成尽调、签约、打款 $500K。</Card>
</CardGrid>

<CardGrid cols={2}>
  <Card title="服务节点 C" icon="⚖️">归因：18%。理由：法律意见、协议审核、合规确认。</Card>
  <Card title="Agent 贡献" icon="🤖">归因：12%。理由：Research Agent 的项目摘要被资本节点采纳作为决策依据（有日志证明），Execution Agent 的纪要和催办推动了流程效率。</Card>
</CardGrid>

<Callout type="info">
归因比例不是固定模版，而是基于证据链条动态计算。如果 Research Agent 的报告没有被采纳，Agent 归因为 0%——只有被采纳并进入闭环的输出才有价值。
</Callout>

## 对比：没有 WCN 时同样的事情怎么做

<CardGrid cols={2}>
  <Card title="没有 WCN" icon="⊘">项目方在 Token2049 发 Deck 给 20 个人 → 其中 5 个回复 → 3 个安排会议（2-3 周协调时间）→ 会议后等反馈（1-2 周）→ 条款谈判通过邮件和 Telegram（1-2 周）→ 法务各自找律所（1-2 周）→ 签约打款（1 周）→ 没有系统记录谁引荐了谁、谁推动了什么。**典型周期：6-10 周。** 且所有过程信息散落在 Telegram、Email 和 Google Docs 中，无法追溯和结算。</Card>
  <Card title="使用 WCN" icon="✅">项目录入即结构化 → Agent 2 小时完成初筛和匹配 → Deal Room 让所有参与方同步信息 → 法务和融资并行推进 → Execution Agent 消除催办和信息同步时间 → 证据自动归集 → 归因自动计算。**典型周期：14 天。** 所有过程证据完整保留，归因清晰，可直接进入结算。</Card>
</CardGrid>

> **效率提升的来源**：不是 WCN 让投资人更快做决策（投资判断需要的时间不会变），而是消除了**协调摩擦**——信息同步、催办跟进、材料整理、法务协调这些「协调税」在传统流程中占 30-40% 的时间，WCN 通过结构化工作流和 Agent 辅助把这部分压缩到接近零。

## 这个案例里，系统得到了什么

<CardGrid cols={2}>
  <Card title="项目方" icon="🎯">14 天完成 $900K 首关 + 法务结构，效率是传统路径的 3-4 倍。所有协作记录完整，可用于后续轮次的信任传递。</Card>
  <Card title="资本方" icon="📊">结构化资料让尽调成本降低 50%+。投资记录进入系统，累积投资人信誉和匹配精度。</Card>
  <Card title="服务方" icon="📋">法务交付被正式记录和归因。不再是「帮忙看了一下」，而是有明确结算依据的正式贡献。</Card>
  <Card title="网络" icon="💎">一个完整的 PoB 闭环进入系统。参与方的信誉评分更新，Agent 的匹配模型获得新的训练数据，未来类似 Deal 的效率会进一步提升。</Card>
</CardGrid>

<Callout type="tip">
这就是 WCN 和介绍群、CRM 或单点工具的本质区别：它让一次协作不只做成结果，还变成系统资产——每一个闭环都让网络变得更强。
</Callout>
