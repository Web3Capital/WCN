## 第四章不讲概念，而是讲动作顺序。

WCN 的工作流不是围绕“热度”设计的，而是围绕“如何让资源变成结果，再让结果进入长期价值层”设计的。

## WCN 的最小逻辑，不是“谁参与过”，而是“谁推动了闭环成立”。

这也是为什么第四章必须出现：它把 WCN 从理论，转成了连续的动作链。

<Callout type="warning">
节点引入资源 → 机会进入 Deal / Task → 人与 Agent 执行 → 结果形成 → 证据进入 Proof Desk → 生成 PoB → 进入结算层
</Callout>

## 本章包含 4 个核心页面。

建议先看最小业务闭环，再看从节点到结算的流程，然后看人和 Agent 如何分工，最后用一个示例把全过程串起来。
