## One logic chain: each part amplifies the others

WCN does not bolt four products together. It runs node, Agent, Proof of Business (PoB), and settlement on a single logic chain. Each part raises the value of the other three.

<CardGrid cols={2}>
  <Card title="Node = responsibility anchor" icon="user">Each added node is a trusted route and a new class of executable task. One more node enriches the supply side for everyone else.</Card>
  <Card title="Agent = amplifier" icon="bot">Agents turn a verified method into a repeatable execution template, so individual experience becomes a network-level capability.</Card>
  <Card title="PoB = value entrance" icon="circle-check">PoB writes who contributed to a result into the record. Matching and attribution sharpen, which draws the next nodes and capital in.</Card>
  <Card title="Settlement = bearing layer" icon="coins">Fees, sharing, and payout align with the same loop, so services and incentives reinforce each other instead of working against each other.</Card>
</CardGrid>

<Callout type="info">
The value of a new participant is higher than the value of a marginal user on an isolated platform. The reason is the cross-side effect: capital, project, service, and node each strengthen the others.
</Callout>

## The advantage is not being first to combine

Being first to assemble these parts is not the moat. Any competitor can copy an interface or restate the concept. The advantage is what accumulates only with time and real transactions.

<CardGrid cols={3}>
  <Card title="Data accumulation" icon="database">Task state, evidence packages, attribution rules, and node performance build up in one semantic layer. A competitor can copy the concept, not the calibrated record of loops that already happened.</Card>
  <Card title="Supply-side exclusivity" icon="lock">Trusted nodes, their deal flow, and the contribution map are bound to PoB. Re-creating that supply elsewhere means rebuilding trust and the sharing rules from zero.</Card>
  <Card title="Compliance barrier" icon="scale">Attribution, evidence standards, and settlement paths encoded for real jurisdictions raise the cost of entry. The barrier grows as the record grows.</Card>
</CardGrid>

<Callout type="warning">
Early on, network data is thin and the barrier is not thick on the first day. The model's strength is that each loop adds to the record by design, rather than resting on a one-time data claim.
</Callout>

## When switching costs become significant

The barrier shows up as switching cost once an organization runs its work on WCN. Migration then means rebuilding the ledger, not flipping a switch.

<Steps>
  <Step title="Reputation and attribution migrate slowly">Contribution recorded in PoB does not transfer. Moving elsewhere requires re-establishing trust and the sharing rules.</Step>
  <Step title="Workflows bind to operations">Agent tasks and approval chains embedded in daily work carry retraining and failure costs when replaced.</Step>
  <Step title="Settlement practices set defaults">Once fee, sharing, and onchain settlement paths run smoothly, finance and legal resist opening a second set of accounts.</Step>
</Steps>

<Statement>Model advantage = one logic chain where each part amplifies the others + data, supply-side exclusivity, and a compliance barrier that accumulate over time. PoB is the value entrance, the Agent is the amplifier, and the node is the responsibility anchor.</Statement>
