# 에이전트는 식별 가능하고, 제한 가능하고, 업그레이드 가능하고 취소 가능해야 합니다.

## 심각한 상태

<CardGrid cols={3}>
  <Card title="등기" icon="file-text">카탈로그에 등록됩니다(이름, 유형, 기본 모델, 소유자 노드, 사용 설명). 비즈니스 데이터가 아직 수신되지 않았습니다. 이는 "계정이 생성되었지만 권한이 활성화되지 않았습니다"와 같습니다.</Card>
  <Card title="인정 받은" icon="key">데이터 도메인, 도구 화이트리스트, 일일 통화 제한, 외부 시스템에 대한 쓰기 허용 여부 등 권한 검토를 완료합니다. 엔터프라이즈 IAM의 역할 바인딩에 해당합니다.</Card>
  <Card title="활동적인" icon="play">승인 범위 내에서 작업을 수행하고 구조화된 로그를 **지속적으로** 작성합니다(입력 요약, 출력 해시, 도구 호출, 승인 결과).</Card>
  <Card title="일시중지됨/일시중지됨" icon="pause">트리거 조건의 예: 비정상적인 무단 시도, 출력 품질 샘플링의 연속 실패, 보안 사고 및 규정 준수 금지. 일시 중지하면 한 번의 클릭으로 도구와 외부 콜백이 차단됩니다.</Card>
  <Card title="업그레이드됨" icon="arrow-up">모델 버전, 프롬프트, 도구 또는 변경 제한 변경 주문 번호, 승인자, 유효 시간 및 롤백 패키지를 기록해야 합니다. 변경 관리(예: ITIL)를 통해 MLOps를 벤치마킹합니다.</Card>
  <Card title="은퇴" icon="flag">해당 업체가 오프라인 상태이거나 기준을 충족하지 못하는 경우 해당 업체는 폐지됩니다. 읽기 전용 로그 보존 기간은 유지되며 학습에 사용되는 민감한 필드는 정책에 따라 삭제되거나 익명화됩니다.</Card>
</CardGrid>

## 라이프 사이클 경로

<Callout type="info">
등록 → 위험 평가 및 승인 → 온라인(Active) → 지속적인 모니터링 및 무작위 검사 → 이벤트 기반 정지 → 변경 업그레이드(승인 있음) → 폐기 및 유지 전략
</Callout>

<Steps>
  <Step title="입장">비즈니스 시나리오, 데이터 분류, PII/기밀유지 조건 처리 여부를 명확히 합니다. "먼저 들어가서 나중에 해결책을 찾는 것"을 금지하십시오.</Step>
  <Step title="승인하다">최소 권한: 에이전트 유형별로 API 범위를 분할합니다. 민감한 작업 또는 HITL(Human in the Loop)에 대한 2인 규칙입니다.</Step>
  <Step title="달리다">SLA: 대기 시간, 오류율 및 환각에 대한 고객 불만 사항 라벨 LangSmith와 같은 관찰 가능성 도구와 유사한 지표이지만 결과는 WCN 작업 시스템에 입력되어야 합니다.</Step>
  <Step title="검토">정기적인 즉석 점검: 동일한 입력을 가진 여러 버전의 모델 비교, 레드팀 프롬프트 샘플, 도구 호출이 범위를 벗어났는지 여부.</Step>
  <Step title="그만두다">은퇴한 요원은 더 이상 파견 목록에 표시되지 않습니다. 과거 출력은 여전히 ​​Proof에서 참조할 수 있지만 모델 버전으로 표시됩니다.</Step>
</Steps>

<Statement>수명 주기 설계 목표: **언제든지 응답** - 실행 중인 에이전트, 사용되는 모델, 액세스할 수 있는 항목 및 마지막 변경을 승인한 사람.</Statement>

## 거버넌스 및 위험 관리 프레임워크(요약)

| 치수 | 핵심 포인트 |
| --- | --- |
| 소유권 | 각 에이전트에는 소유 노드 또는 플랫폼 담당자가 있습니다. 운행 중 사고가 발생하면 먼저 소유자에게 연락하세요 |
| 변경 | 프로덕션 변경 사항은 변경 창과 롤백을 거칩니다. 주요 모델 전환에는 병렬 회색조가 필요할 수 있음 |
| 데이터 | 훈련/로그 보유 및 국경 간 전송은 노드 위치 및 계약을 준수합니다 |
| 보안 | 키 및 도구 자격 증명의 에스크로 에이전트가 사용자 일반 텍스트 비밀번호를 지속적으로 보유하는 것을 금지 |
| 감사 | 로그 보존 기간 ≥ 비즈니스 분쟁 추적 요구 사항 제3자 실사로의 수출 지원 |

<Callout type="warning">
암묵적으로 일시 중지하고 은퇴할 수 있는 능력이 없기 때문에 "좀비 에이전트"가 만료된 권한을 오랫동안 보유할 수 있습니다. 이는 데이터 위반 및 규정 준수 처벌의 일반적인 원인입니다.
</Callout>

## AutoGPT 스타일의 "상시 개방형 루프"와의 근본적인 차이점

자율 루프의 기본값은 **명확한 끝점 없음**입니다. WCN 수명 주기는 각 실행을 **작업 및 인증 만료**와 연결합니다. 작업이 종료되거나 권한이 만료됩니다. → 에이전트 호출은 백그라운드에서 계속 "생각"하는 대신 거부되거나 읽기 전용으로 입력되어야 합니다.

<Callout type="tip">
감사자나 LP에게 설명할 때 한 문장: **저희 에이전트는 지치지 않는 자동화 작업자가 아니라 통제된 배치 작업과 같습니다. **
</Callout>
