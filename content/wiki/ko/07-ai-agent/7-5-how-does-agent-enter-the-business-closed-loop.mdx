# 채택과 증명은 가치의 관문입니다. 세대는 그렇지 않습니다.

## 폐쇄 루프로의 주요 경로

<Callout type="info">
작업 파견(유형 및 수락 기준 포함) → 에이전트가 승인 내에서 실행 → 구조화된 출력 저장 → 노드 소유자 채택/거부/수정 → 상태 머신에서 거래 또는 작업 진행 → 결과 증거 제출 → 증명 데스크 검토 → PoB 기록 생성 → 결제 및 평판 레이어 입력
</Callout>

<Statement>에이전트의 가치는 식별 가능한 **결과 동인**입니다. 시스템은 "10페이지가 생성되었습니다"와 "그 중 3페이지가 채택되어 회의 개최/조건 수정/지불 조건 충족"을 구분할 수 있습니다.</Statement>

## Agent 출력이 Proof 입력에 필요한 조건

<CardGrid cols={2}>
  <Card title="작업 ID 바인딩" icon="link">각 기계 출력은 `task_id`(또는 이에 상응하는 핸들)와 연결됩니다. 비공개 채팅이나 오프라인 문서에서 배회하는 "유령 결과"는 금지됩니다.</Card>
  <Card title="입력 및 버전 추적" icon="clipboard-list">모델 이름, 버전, 온도 또는 이에 상응하는 매개변수, 검색 조각 ID, 키 첨부 해시 분쟁 발생 시 복제 및 증거 수집을 용이하게 합니다.</Card>
  <Card title="인간 입양 기록" icon="circle-check">명시적 이벤트: `adopted_by`, `timestamp`, 선택적 수정 diff; Copilot의 암시적 "사용자 복사"와는 달리 구조화되어야 합니다.</Card>
  <Card title="결과 상관관계" icon="refresh-cw">이는 거래, 이정표 또는 온체인/오프체인 비즈니스 이벤트(예: 계약서 제출, 증권 거래소 발표)로 추적될 수 있습니다. 그렇지 않으면 귀속정산을 할 수 없습니다.</Card>
</CardGrid>

<Callout type="warning">
조건 중 하나라도 누락된 경우 → "도구 생성"으로 분류되어 PoB에 포함되지 않습니다. TradFi의 "거래 번호 및 작업 서류가 없는 이메일"과 같은 이유로 계정에 포함될 수 없습니다.
</Callout>

## 세 가지 구체적인 작업흐름 예시

<Steps>
  <Step title="자금 조달 반복">작업: "시리즈 A 재료 v3 위험 장". 연구 에이전트는 경쟁 제품 위험 공개 가져오기 → 참조가 포함된 초안 출력 → 노드 삭제 및 채택 → 수정된 PDF 해시 업로드 → 증명 동료 "투자자 회의 최종 초안" → 회의 종료 후 PoB 기록.</Step>
  <Step title="자본 매칭">Deal Agent는 상위 5개의 항목과 일치 이유를 출력합니다. → 노드는 그 중 2개만 채택하고 소개 작업을 생성합니다. → 후속 이메일은 사람에 의해 전송됩니다. PoB는 채택된 추천과 후속 검증 결과만 기록하며, 채택되지 않은 순위는 기록하지 않습니다.</Step>
  <Step title="투자 후 모니터링">Liquidity Agent에서 비정상적인 문단 보고 → Execution Agent에서 내부 Slack/Task Copy 초안 생성 → 리스크 관리 담당자가 이를 채택하여 "Follow-up Custody Bank" Task 생성 → 영수증이 증거 패키지에 입력됨 → Proof가 통과된 후 PoB로 전달됩니다.</Step>
</Steps>

## 반례: 매우 바쁜 것 같고 폐쇄 루프가 없습니다.

- AutoGPT 스타일의 무제한 탐색은 많은 수의 메모를 생성하고 작업도 없고 소유자도 없습니다 → **노이즈**.
- Copilot은 예쁜 이메일을 생성하지만 WCN 작업에서 전송되지 않으며 채택 태그가 없습니다 → **귀속 불가**.
- 에이전트가 타임라인으로 이동을 제안했고 노드는 이를 프로젝트 당사자에게 구두로 전달했지만 시스템은 상태 비저장 → **검증할 수 없음**.

## 이 단계는 무엇을 결정합니까?

<CardGrid cols={2}>
  <Card title="AI는 도구일 뿐인가?" icon="bot">폐쇄 루프에 들어간 후에만 공식적인 실행 레이어가 됩니다. 그렇지 않으면 LLM 패키징과 다르지 않습니다.</Card>
  <Card title="Agent 정산 가능 여부" icon="coins">증명 체인이 없으면 PoB도 없습니다. 인센티브와 이익 공유에는 자격이 부족합니다.</Card>
  <Card title="에이전트를 관리할 수 있나요?" icon="shield">특정 에이전트 구성을 업그레이드하거나 일시 중단하기 전에 채택률, 오류 채택 롤백 및 모델 A/B 효과를 모두 측정할 수 있습니다.</Card>
  <Card title="WCN 차별화" icon="zap">"GPT/Claude/Gemini에 연결"되지는 않지만 다중 노드 비즈니스 네트워크에서 기계 출력이 통합 증거 및 결제 의미 체계에 연결되는 것은 처음입니다.</Card>
</CardGrid>

<Callout type="tip">
승인 기준에 대한 제안: **분쟁 중에 로그를 사용하여 30분 안에 명확하게 설명할 수 있습니까** - 누가 에이전트에게 무엇을 요청했는지, 무엇을 채택했는지, 최종 비즈니스 결과는 무엇인지. 이것이 가능하다면 폐쇄 루프 설계는 기본적으로 적합합니다.
</Callout>
