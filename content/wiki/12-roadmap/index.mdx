# 先证明商业网络成立，再叠加链与资产复杂度——与以太坊分阶段扩容、Uniswap 从 v1 到 v3 的演进逻辑一致。
</Hero>

<MetaGrid>
  <Meta label="核心主题" value="Network MVP → PoB 协同 → 结算资产层" />
  <Meta label="方法论" value="业务优先、链后置（business first, chain later）" />
  <Meta label="适用对象" value="投资人、战略、合作方、核心节点" />
</MetaGrid>

<Callout type="info">典型「一次性上链 + Token + 全功能」路线图往往在主网前耗尽叙事弹性；分阶段交付把每一阶段的可验证产出（用户、收入、标准、风控）暴露给市场，与 Solana 生态先跑应用再补基础设施节奏、以太坊以 Rollup 为中心的分阶段路线图同属一类可信叙事。</Callout>

<Steps>
  <Step title="第一阶段 · Network MVP">节点准入、Deal Room 最小流程、Proof Desk 留痕、早期单位经济（席位/服务/成交费）可计量。</Step>
  <Step title="第二阶段 · PoB + Agent">PoB 规则与审核稳定；Agent 权限、日志、生命周期可控；高价值打法流程化，闭环密度可复盘。</Step>
  <Step title="第三阶段 · 结算与资产">在 PoB 成为可信输入的前提下，推进结算机制与身份/价值的数字化承载；链上仅承载必要摘要与证明。</Step>
</Steps>
