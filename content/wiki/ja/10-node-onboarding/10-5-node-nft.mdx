<Hero badge="10.5 · ノードNFT" lead="ノード NFT の主な用途は、プログラム可能な ID と権限の保持です。ウォレットには、配当権利証明書やオープンキャスト PFP ではなく、「特定バージョンのルールに基づく特定タイプのシートへのシステム入口」が保持されます。この設計思想は、プロフェッショナルなアクセス コントロール カード (物理) と SAML/OIDC 属性アサーション (従来の IT) を利用していますが、オンチェーン オブジェクトを使用してアプリケーション間の結合可能な検証を実現します。">

## 一般的な 3 種類の ID テクノロジーとの比較

|ソリューション |強み |制限事項 |ノードNFTの選択肢 |
| --- | --- | --- | --- |
| **ENS** |人間が判読可能、広く統合されている |複雑な権限や組織関係をネイティブに表現しない。 ENS と並べて使用できます。ENS は人物/ブランドを指し、ノード NFT は **機関席**を指します。
| **SBT (ソウル バインディング トークン)** |非譲渡性と評判拘束を重視 |アップグレード、シートの移行、およびコンプライアンス凍結のシナリオは柔軟性に欠けます。 WCN は、**ポリシーによって決定される譲渡可能性** を持つ証明書を使用できます (ロック可能、格納可能なマッピング)。
| **W3C 検証可能な資格情報** |標準プライバシーモデル、選択的開示 |弱いオンチェーンのコンポーザビリティとウォレットのネイティブ表示 |重要なステートメントは二重に記述される可能性があります。VC はコンプライアンスの開示に使用され、NFT は **プロトコル内の高速アクセス制御**に使用されます。

<CardGrid cols={2}>
  <Card title="IDキャリア" icon="id-card">このチェーンは、「特定のアドレスが特定のノード サブジェクトおよびシート世代にバインドされている」ことを証明します。リフレッシュ権限は、ウォレットを頻繁に変更するのではなく、メタデータのバージョンまたはサイドチェーンの状態ルートを介して渡されることがよくあります。</Card>
  <Card title="許可事業者" icon="key">API を呼び出す、コントラクト関数を入力する、またはフロントエンド ルーティングのロックを解除するための **必要だが十分ではない** 条件として。最終的には、ハードウェア トークン + サーバー セッションと同様に、サーバー ACL とコンプライアンス フラグが優先されます。</Card>
  <Card title="記録媒体（進化）" icon="chart-bar">レピュテーション スコア、レベル、および履歴タスク ハッシュ ポインターをマウントできます。完全な証拠は、メインネットのガスとプライバシーの問題を避けるために、オフチェーンのオブジェクト ストレージとプルーフ デスクにまだほとんど保管されています。</Card>
  <Card title="プログラマビリティ" icon="link-2">サポートおよびインセンティブ契約、共同マーケティング ホワイトリスト、クロスパートナー SDK ドッキング。制限はスマート コントラクト層で行う必要があります** ノード NFT は有価証券または預金証書として表現することはできません**。</Card>
</CardGrid>

## NFT が明示的にそうでないノード

<CardGrid cols={2}>
  <Card title="金融商品ではありません" icon="ban">経営規模に応じた元本、固定収入、または自動利益分配の約束はなく、認可されたファンド/預金の法的境界線から明確に分離されています。</Card>
  <Card title="純粋なアバターではない" icon="ban">ビジュアルレイヤーは不要です。この価値は、WCN アカウント、契約義務、および KPI との拘束関係から生まれます。</Card>
  <Card title="ライセンスのないキャスト" icon="ban">キャスト権限はネットワークのガバナンス/運用側にあります。ミントを開放することは座席の不足とリスク管理の前提を破壊することになる。</Card>
  <Card title="チェーン上のマスターキーではありません" icon="ban">秘密キーが紛失した場合、主体が契約を終了した場合、またはコンプライアンスがトリガーされた場合は、マッピングの凍結、アドレスのローテーション、世代キーの無効化などの運用および保守方法を通じて対処する必要があります。</Card>
</CardGrid>

## 実装原則

<Callout type="warning">**ビジネスネットワークファースト**: まず実際の取引、タスク、レビュー担当者、証拠の基準を設け、次に NFT の適用範囲を拡大します。そうしないと、「まず会員カードを販売してからクラブを改装する」のと同じリスクが発生し、評判と監督が圧迫されることになります。</Callout>

<Statement>現段階では、ノード NFT は、チェーン上の投機的なターゲットではなく、**プロフェッショナル ネットワーク用のデジタル カード**として理解されるべきです。つまり、チェーン上で検証可能であり、チェーン外では説明責任があり、システムはワンクリックで回復できます。</Statement>
