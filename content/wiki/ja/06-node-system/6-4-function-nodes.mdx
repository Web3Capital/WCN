# ファンクションノードの答え: クローズドループでは、資産側、資本側、受け渡し側、市場側、または実行アンプを提供しますか?
</Hero>

  <Meta label="閉ループシーケンス" value="プロジェクト定義要件 → 資本価格設定と構造化 → サービス実装 → 配布と受け入れ → エージェントの大規模な反復労働" />
  <Meta label="TradFi の例え" value="企業財務、販売およびトレーディング、オペレーション、マーコム、およびミドルオフィスオートメーションの大まかなマッピング" />
  <Meta label="ガバナンスのポイント" value="関数タグはデフォルトの PoB テンプレートと KPI にバインドされています。部門をまたがる注文には事前設定されたルールが必要" />
</MetaGrid>

## 機能ノードの概要

<CardGrid cols={2}>
  <Card title="プロジェクトノード" description="たとえば、L2、RWA プラットフォーム、決済アプリケーション: ロードマップ、コンプライアンス戦略、資金調達リズム、通貨リスト要件を実行します。それらは需要と資産の物語の源です。" icon="package" />
  <Card title="首都ノード" description="たとえば、単一企業の割り当て範囲が 500 万米ドルから 5,000 万米ドルである場合や、シードに焦点を当てた暗号ファンドなど、評価ロジック、タームシート、および共同投資リズムを決定します。" icon="coins" />
  <Card title="サービスノード" description="たとえば、一流の法律事務所のトークン グループ、スマート コントラクトの監査人、ブランド広報担当者などは、口頭での推奨事項を署名ページや成果物に変えます。" icon="wrench" />
  <Card title="配信ノード" description="たとえば、Tier1 CEX 通貨リスト オフィス、主要な KOL マトリックス、大規模なサミット IP: TGE の引き受け、マーケット メイキング パッケージおよびコミュニケーション スケジュールなどです。" icon="radio" />
  <Card title="地域ノード" description="国家マスターノードと都市ノード: 活動、政府関係、地域資本を地理的次元で組織します。詳細については、6.5 および 6.6 を参照してください。" icon="globe" />
  <Card title="エージェントノード" description="たとえば、資金調達ラウンドの 7 時間 365 日のモニタリングや、最初のラウンドの連絡メールの自動作成、つまりモデルの境界、手動レビュー ポイント、ログ保存戦略などを開示する必要があります。" icon="bot" />
</CardGrid>

<Statement>まず機能を分割し、次に責任と貢献度の測定を分割します。そうしないと、異なるリンクにおける同じ組織の役割が混在し、PoB は紹介料、実行料、成功料を分離できなくなります。</Statement>

## 機能分業とTradFiテーブルトップの比較

<CardGrid cols={2}>
  <Card title="プロジェクトノード" description="法人アカウント部門に対応: 取引の動機とスケジュールを定義します。" icon="target" />
  <Card title="首都ノード" description="資本市場への買い手の販売に対応する: 製品と制約のマッチング。" icon="coins" />
  <Card title="サービスノード" description="ミドルオフィスとサードパーティのコンサルタントに対応: 用語をドキュメントとコードに記述します。" icon="wrench" />
  <Card title="配信ノード" description="ECM/販売取引インターフェースに対応: ストーリーを注文フローと流動性に変換します。" icon="radio" />
  <Card title="地域ノード" description="対応する支店の対応範囲: ローカライズされた関係と規制に関するコミュニケーション。" icon="globe" />
  <Card title="エージェントノード" description="対応する定量化および中間事務スクリプト: 標準プロセスを強化しますが、規制された判断の開示点に代わるものではありません。" icon="bot" />
</CardGrid>

<Callout type="tip">
地域ノードは、6.3 の概要では機能と並んで表示されますが、多くの場合、「機能 + 地理」の重ねられたラベルとして実装されます。 6.4 では、純粋なオンライン資本やサービス ノードとの混同を避けるために、その機能セマンティクスが「ローカルな組織と実装」であることが強調されています。
</Callout>
