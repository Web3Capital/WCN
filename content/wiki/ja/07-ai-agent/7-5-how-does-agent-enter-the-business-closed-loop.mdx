## 閉ループへの主なパス

<Callout type="info">
タスクのディスパッチ（タイプおよび受け入れ基準を含む） → エージェントが承認内で実行 → 構造化された出力が保存される → ノード所有者が採用/拒否/修正 → ステートマシンが取引またはタスクを進める → 結果証拠の提出 → プルーフデスクレビュー → PoB 記録の生成 → 決済および評判層に入る
</Callout>

<Statement>エージェントの値は、識別可能な **結果ドライバー** です。システムは、「10 ページが生成された」と「そのうちの 3 ページが採用され、会議が開催された/条件が変更された/支払い条件が満たされた」を区別できます。</Statement>

## エージェントの出力が Proof に入るために必要な条件

<CardGrid cols={2}>
  <Card title="タスクIDのバインド" icon="link">各マシンの出力は「task_id」（または同等のハンドル）に関連付けられます。プライベート チャットやオフライン ドキュメント内を徘徊する「ゴースト結果」は禁止されています。</Card>
  <Card title="入力とバージョンのトレース" icon="clipboard-list">モデル名、バージョン、温度または同等のパラメータ、取得フラグメント ID、キー添付ハッシュ。紛争の際の複製と証拠収集を容易にするため。</Card>
  <Card title="人間の養子縁組記録" icon="circle-check">明示的なイベント: `adopted_by`、`timestamp`、オプションの変更差分。 Copilot の暗黙的な「ユーザーがコピーされた」とは異なり、構造化する必要があります。</Card>
  <Card title="結果の相関関係" icon="refresh-cw">それは、取引、マイルストーン、またはオンチェーン/オフチェーンのビジネス イベント (契約の提出、証券取引所の発表など) まで遡ることができます。そうしないと、帰属の決済を行うことができません。</Card>
</CardGrid>

<Callout type="warning">
いずれかの条件が欠落している場合 → 「生成されたツール」として分類され、PoB には含まれません。 TradFiの「取引番号や調書が記載されていないメール」と同様の理由で口座に含めることはできません。
</Callout>

## 3 つの具体的なワークフロー例

<Steps>
  <Step title="資金調達材料の反復">タスク: 「シリーズ A マテリアル v3 リスクの章」。調査エージェントは競合製品のリスク開示情報を取得 → 参照付き草案を出力 → ノードの削除と採用 → 修正された PDF ハッシュのアップロード → 「投資家会議の最終草案」を証明 → 会議終了後に PoB を記録。</Step>
  <Step title="資本マッチング">Deal Agent は上位 5 位のアイテムと一致する理由を出力します。 → ノードはそのうちの 2 つだけを採用し、導入タスクを作成します。 → 後続の電子メールは人間によって送信されます。 PoB は採用された推奨事項とその後の検証結果のみを記録し、採用されなかったランキングは記録しません。</Step>
  <Step title="投資後のモニタリング">流動性エージェントが異常な段落を報告 → 実行エージェントが社内の Slack/タスク コピー ドラフトを生成 → リスク管理担当者がそれを採用し、「フォローアップ保管銀行」タスクを作成 → 領収書が証拠パッケージに入力 → 証拠が渡され、PoB に転送されます。</Step>
</Steps>

## 反例: 非常に忙しいようです、ゼロ閉ループ

- AutoGPT スタイルの無制限の探索では、大量のメモが生成され、タスクも所有者も存在しません → **ノイズ**。
- Copilot はきれいな電子メールを生成しますが、WCN タスクから送信されず、採用タグがありません → **属性なし**。
- エージェントはタイムラインに移動することを提案し、ノードはそれを口頭でプロジェクト パーティに転送しましたが、システムはステートレス → **検証できません**。

## このステップで何が決まるのでしょうか?

<CardGrid cols={2}>
  <Card title="AIは単なるツールでしょうか？" icon="bot">閉ループに入った後にのみ正式な実行層になります。それ以外の場合は、LLM パッケージと何ら変わりません。</Card>
  <Card title="エージェントが決済できるかどうか" icon="coins">証明チェーンがなければ、PoB は存在しません。インセンティブと利益分配には資格がありません。</Card>
  <Card title="エージェントは管理可能ですか?" icon="shield">導入率、導入エラーのロールバック、モデル A/B の効果はすべて、特定のエージェント構成をアップグレードまたは一時停止する前に測定できます。</Card>
  <Card title="WCNの差別化" icon="zap">「GPT/Claude/Gemini に接続されている」わけではありませんが、マシンの出力が統一された証拠と決済セマンティクスに接続されるのは、マルチノード ビジネス ネットワークでは初めてです。</Card>
</CardGrid>

<Callout type="tip">
受け入れ基準に関する提案: **紛争中にログを使用して 30 分で明確に説明できますか** - 誰がエージェントに何を依頼したか、何が採用され、最終的なビジネス結果はどうなったか。それができれば、閉ループ設計は基本的に適格です。
</Callout>
