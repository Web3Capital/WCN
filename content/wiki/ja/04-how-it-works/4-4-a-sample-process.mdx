## 14日間の完了手続き

### 1 ～ 2 日目: リソースの入力とマッチング

<CardGrid cols={1}>
  <Card title="1日目: プロジェクトエントリー" icon="inbox">プロジェクト ノードは、デッキ、トークンノミクス ドラフト、チームの背景、技術文書、資金調達条件 (200 万ドルの SAFE + トークン ワラント) の情報を Project Intake に送信します。 Research Agent は、構造化された概要 + リスク リスト + 競合比較を 2 時間で生成します。</Card>
  <Card title="2日目: マッチングとディールルームの作成" icon="handshake">Deal Agent はキャピタル ノードの設定に基づいて A と B を照合します。システムはディール ルームを作成し、ディール オーナー (推奨される地域ノードが演じる) が指定され、サービス ノード C が法的トラックに追加されます。すべての関係者が同じ構造化情報を参照します。</Card>
</CardGrid>

### 3 ～ 7 日目: 実行の進歩

<CardGrid cols={1}>
  <Card title="3日目: 最初のミーティング" icon="phone">取引オーナーは、プロジェクト関係者と資本ノード A の間でビデオ会議を開催します。実行エージェントはリアルタイムで議事録を生成し、5 つのアクション アイテムを自動的に抽出し、オーナーを割り当てます。会議の 30 分以内に、すべての参加者は構造化された議事録を受け取ります。</Card>
  <Card title="4 ～ 5 日目: デューデリジェンスと法務業務を並行して行う" icon="zap">Capital Node A には、補足的な技術監査レポートと創設者の経歴調査が必要です。 Research Agent は 2 時間でそれを解決しました。同時に、サービス ノード C は SAFE + トークン保証条件の検討を開始し、5 日目に予備的な法的意見を発行します。</Card>
  <Card title="6 日目: 第 2 ラウンド会議 + 首都ノード B エントリー" icon="phone">Capital Node A はフォローアップの意向を確認します (40 万ドル)。 Capital Node B は、情報を読んだ後、別の会議を要求しました。 Deal Room は 2 つの資本ラインを同時に管理し、ステータスは個別に追跡されます。</Card>
  <Card title="7日目: 条件交渉" icon="clipboard-list">プロジェクト当事者とキャピタルノードAは、評価額とトークンワラント比率について合意に達します。サービス ノード C は最終的な法的意見を発行します。実行エージェントは、署名されるすべてのファイルのステータスを追跡します。</Card>
</CardGrid>

### 8 ～ 11 日目: 契約書の署名と支払い

<CardGrid cols={1}>
  <Card title="8～9日目: 契約書への署名" icon="pen-line">Capital Node A は SAFE ($400,000) + トークンワラントに署名します。 Capital Node B は SAFE ($500,000) + トークンワラントに署名します。サービス ノード C は、すべてのプロトコルのコンプライアンス レビューを完了します。</Card>
  <Card title="10～11日目: 支払い確認" icon="banknote">首都ノード A は 40 万ドル (銀行振込) の支払いを完了しました。 Capital Node B は 500,000 ドル (300,000 ドルのワイヤ + 200,000 ドルの USDC) の送金を完了しました。実行エージェントは支払いステータスを追跡し、銀行確認をオンチェーン記録に自動的に関連付けます。</Card>
</CardGrid>

### 12 ～ 14 日目: 証拠収集と PoB

<CardGrid cols={1}>
  <Card title="12日目: 証拠の梱包" icon="paperclip">取引所有者は、次の証拠を証明デスクに提出します: 署名された SAFE 契約書 (2 部)、トークンワラント (2 部)、支払銀行確認 (2 部)、オンチェーン USDC 転送ハッシュ (1 部)、法的意見 (1 部)、およびすべての会議議事録 (4 部)。</Card>
  <Card title="13 日目: 復習" icon="search">独立した査読者が証拠の完全性と信頼性をチェックします。確認: 資金調達額は 90 万ドル (このラウンドの最初のレベル)、契約条件はエントリーと一致しており、法的な納品は完了しており、スケジュールは妥当です。</Card>
  <Card title="14 日目: PoB 生成 + アトリビューション" icon="circle-check">レビューに合格すると、システムは PoB イベントを生成します。帰属は次のように割り当てられます（下記を参照）。</Card>
</CardGrid>

## アトリビューション割り当ての詳細

<CardGrid cols={3}>
  <Card title="取引オーナー (リージョンノード)" icon="crown">帰属: 30%。理由: プロジェクト パーティーを推奨し、ディール ルームを作成し、すべての会議を開催し、署名の進捗を促進します。</Card>
  <Card title="キャピタルノードA" icon="landmark">帰属: 20%。理由: デューデリジェンスを完了し、契約に署名し、40 万ドルを支払います。</Card>
  <Card title="首都ノードB" icon="landmark">帰属: 20%。理由: デューデリジェンスを完了し、契約書に署名し、50 万ドルを支払います。</Card>
</CardGrid>

<CardGrid cols={2}>
  <Card title="サービスノードC" icon="scale">帰属: 18%。理由: 法的意見、契約書レビュー、コンプライアンス確認。</Card>
  <Card title="エージェントの貢献" icon="bot">帰属: 12%。理由: リサーチ エージェントのプロジェクト概要は、意思決定の基礎としてキャピタル ノードによって採用され (ログによって証明されます)、実行エージェントの議事録とリマインダーはプロセスの効率を促進します。</Card>
</CardGrid>

<Callout type="info">
アトリビューション率は固定テンプレートではなく、証拠チェーンに基づいて動的に計算されます。研究エージェントのレポートが採用されない場合、エージェントの帰属は 0% になります。採用されて閉ループに入った出力のみが価値があります。
</Callout>

## 比較: WCN を使用せずに同じことを行う方法

<CardGrid cols={2}>
  <Card title="トイレなし" icon="ban">プロジェクト当事者が Token2049 の 20 人にデッキを送信 → そのうち 5 人が応答 → 3 回の打ち合わせ（調整に 2 ～ 3 週間） → 打ち合わせ後のフィードバックを待つ（1 ～ 2 週間） → 電子メールと電報による条件交渉（1 ～ 2 週間） → 法務関連の法律事務所を見つける（1 ～ 2 週間） → 契約書への署名と支払い（1 週間） → 誰が誰を推薦し、誰が何を推進したかを記録するシステムはありません。 **一般的な周期: 6 ～ 10 週間。 ** そして、すべてのプロセス情報は Telegram、Email、Google Docs に分散されているため、追跡して解決することは不可能です。</Card>
  <Card title="WCNを使用する" icon="circle-check">プロジェクトエントリーが仕組み化されている → エージェントによる事前審査とマッチングが2時間で完了 → ディールルームで参加者全員が情報を同期 → 法務と資金調達を並行して推進 → 執行エージェントによる催促や情報同期時間の削減 → 証拠の自動収集 → 帰属の自動計算**通常のサイクル時間: 14 日。 ** すべてのプロセス証拠はそのまま保持され、帰属が明確で、和解を直接入力できます。</Card>
</CardGrid>

> **効率向上の源**: WCN によって投資家が意思決定を迅速化できるわけではありません (投資判断に必要な時間は変わりません)。情報の同期、リマインダーのフォローアップ、資料の仕分け、法的調整などの **調整摩擦** が排除されます。従来のプロセスでは、これらの「調整税」が時間の 30 ～ 40% を占めています。 WCN は、構造化されたワークフローとエージェントの支援を通じて、この部分をほぼゼロに圧縮します。

## この場合、システムは何を取得したのでしょうか?

<CardGrid cols={2}>
  <Card title="プロジェクトパーティー" icon="target">90 万ドルの初回パス + 法的構造を 14 日間で完了すると、効率は従来のパスの 3 ～ 4 倍になります。すべてのコラボレーション記録は完了しており、後続の信託譲渡ラウンドに使用できます。</Card>
  <Card title="資本側" icon="chart-bar">構造化データにより、デューデリジェンスのコストが 50% 以上削減されます。投資記録がシステムに入力され、投資家の信頼性と照合精度が蓄積されます。</Card>
  <Card title="サービスプロバイダー" icon="clipboard-list">法的成果物は正式に文書化され、帰属が証明されます。それはもはや「様子見のお手伝い」ではなく、明確な和解根拠を伴う正式な寄付である。</Card>
  <Card title="ネットワーク" icon="gem">完全な PoB 閉ループがシステムに入ります。参加者の評判スコアが更新され、エージェントのマッチング モデルが新しいトレーニング データを取得します。将来的には、同様の取引の効率がさらに向上します。</Card>
</CardGrid>

<Callout type="tip">
これが、WCN と紹介グループ、CRM、またはポイント ツールとの本質的な違いです。これにより、コラボレーションが結果を生み出すだけでなく、システム資産になることも可能になります。それぞれの閉ループがネットワークを強化します。
</Callout>
