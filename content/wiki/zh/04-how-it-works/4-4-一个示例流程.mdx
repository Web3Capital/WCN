## 场景设定

**项目方**：一个 AI 与 DeFi 基础设施项目——团队在新加坡，技术在欧洲——正在融 $2M 种子轮。它需要两件事。

<CardGrid cols={2}>
  <Card title="合适的资本" icon="coins">理解 AI 与 DeFi 交叉领域、接受 SAFE 加 Token Warrant 结构、票面 $200K 到 $500K 的基金。</Card>
  <Card title="完整的法务结构" icon="file-stack">SAFE 协议审核、Token Warrant 条款、新加坡实体合规检查与投资人 KYC 流程。</Card>
</CardGrid>

**WCN 网络中的参与方。**

<CardGrid cols={3}>
  <Card title="资本节点 A" icon="landmark">香港基金，AI 与 Infra 赛道，种子轮票面 $300K 到 $500K。</Card>
  <Card title="资本节点 B" icon="landmark">迪拜家族办公室，DeFi 基础设施，票面 $500K 到 $1M。</Card>
  <Card title="服务节点 C" icon="scale">新加坡法务团队，专长 SAFE、SAFT 与 Token Warrant 结构。</Card>
</CardGrid>

<CardGrid cols={2}>
  <Card title="Research Agent" icon="microscope">把项目数据结构化、运行竞品分析、标记风险，并起草投资备忘录。</Card>
  <Card title="Execution Agent" icon="zap">生成会议纪要、追踪待办、发出提醒，并监控材料收集状态。</Card>
</CardGrid>

## 14 天闭环

### Day 1–2：资源录入与匹配

<CardGrid cols={1}>
  <Card title="Day 1：项目录入" icon="inbox">项目节点提交至 Project Intake：Deck、Tokenomics 草案、团队背景、技术文档与融资条款（$2M SAFE 加 Token Warrant）。Research Agent 在两小时内返回结构化摘要、风险清单与竞品对比。</Card>
  <Card title="Day 2：匹配与 Deal Room" icon="handshake">Deal Agent 依据各自偏好匹配出节点 A 与 B。系统开启 Deal Room，指定 Deal Owner——引荐项目的区域节点，并把服务节点 C 加入法务 Track。所有参与方看到同一份结构化信息。</Card>
</CardGrid>

### Day 3–7：执行推进

<CardGrid cols={1}>
  <Card title="Day 3：首次会议" icon="phone">Deal Owner 召集项目方与资本节点 A 的通话。Execution Agent 实时生成纪要，提取五项带 owner 的行动项。所有参与方在 30 分钟内收到结构化纪要。</Card>
  <Card title="Day 4–5：尽调与法务并行" icon="zap">资本节点 A 要求补充技术审计与创始人背景调查；Research Agent 在两小时内整理完毕。同时，服务节点 C 审核 SAFE 加 Token Warrant 条款，在 Day 5 出具初步法律意见。</Card>
  <Card title="Day 6：第二轮与节点 B 进入" icon="phone">资本节点 A 确认跟进意向，金额 $400K。资本节点 B 看完资料后要求单独会议。Deal Room 同时运行两条资本线，各自分别追踪。</Card>
  <Card title="Day 7：条款" icon="clipboard-list">项目方与资本节点 A 就估值与 Token Warrant 比例达成一致。服务节点 C 出具最终法律意见。Execution Agent 追踪每份待签文件的状态。</Card>
</CardGrid>

### Day 8–11：签署与打款

<CardGrid cols={1}>
  <Card title="Day 8–9：协议签署" icon="pen-line">资本节点 A 签署 SAFE（$400K）加 Token Warrant。资本节点 B 签署 SAFE（$500K）加 Token Warrant。服务节点 C 完成所有协议的合规审核。</Card>
  <Card title="Day 10–11：打款确认" icon="banknote">资本节点 A 电汇 $400K。资本节点 B 打款 $500K（$300K 电汇加 $200K USDC）。Execution Agent 追踪打款状态，并把银行确认与链上记录关联。</Card>
</CardGrid>

### Day 12–14：证据与证明

<CardGrid cols={1}>
  <Card title="Day 12：证据打包" icon="paperclip">Deal Owner 提交至 Proof Desk：已签署的 SAFE 协议（2 份）、Token Warrant（2 份）、银行打款确认（2 份）、链上 USDC 转账哈希（1 笔）、法律意见书（1 份），以及全部会议纪要（4 份）。</Card>
  <Card title="Day 13：复核" icon="search">一名独立复核人检查完整性与真实性，并确认：本轮首关融资合计 $900K、协议条款与录入一致、法务交付完整、时间线一致。</Card>
  <Card title="Day 14：验证节点与归因" icon="circle-check">复核通过。系统在三账本之间对账，在证明账本生成一个验证节点，并按下方分配归因。</Card>
</CardGrid>

## 归因比例

<CardGrid cols={3}>
  <Card title="Deal Owner（区域节点）" icon="crown">比例：30%。引荐项目、开启 Deal Room、召集每一次会议，并推动签约完成。</Card>
  <Card title="资本节点 A" icon="landmark">比例：20%。完成尽调、签约，并打款 $400K。</Card>
  <Card title="资本节点 B" icon="landmark">比例：20%。完成尽调、签约，并打款 $500K。</Card>
</CardGrid>

<CardGrid cols={2}>
  <Card title="服务节点 C" icon="scale">比例：18%。出具法律意见、审核协议，并确认合规。</Card>
  <Card title="Agent 贡献" icon="bot">比例：12%。Research Agent 的摘要被一个资本节点采纳为决策输入，有日志为证；Execution Agent 的纪要与催办推进了流程效率。</Card>
</CardGrid>

<Callout type="info">
归因比例不是固定模版。系统依据证据链条计算。如果一份 Research Agent 报告未被采纳，其比例为零——只有被采纳并进入闭环的产出才带有价值。
</Callout>

## 无 WCN 时的同一件事

<CardGrid cols={2}>
  <Card title="没有 WCN" icon="ban">项目方在一场会议上发 Deck 给 20 个人。五个回复。三个安排会议，协调需要 2 到 3 周。反馈在会议后的 1 到 2 周陆续到来。条款经邮件与 Telegram 推进 1 到 2 周。找律所需要 1 到 2 周。签约与打款用掉一周。没有系统记录谁引荐了谁、谁推进了什么。典型周期 6 到 10 周，记录散落在 Telegram、邮件与文档中——且没有通往结算的路径。</Card>
  <Card title="使用 WCN" icon="circle-check">项目录入即结构化。Agent 在两小时内完成初筛与匹配。Deal Room 让每一方保持同步。法务与融资并行推进。Execution Agent 消除提醒与同步时间。证据被收集，归因被计算。典型周期 14 天，证据链条完整、归因明确、结算就绪。</Card>
</CardGrid>

> **收益从何而来**：WCN 不让投资人决策更快；形成投资判断所需的时间不变。它消除的是协调摩擦——信息同步、催办跟进、材料整理与法务协调。这部分协调税在传统流程中占 30% 到 40% 的时间。结构化工作流与 Agent 支撑把它压向零。

## 系统得到了什么

<CardGrid cols={2}>
  <Card title="项目方" icon="target">在 14 天内完成 $900K 首关加法务结构。每一条协作记录都完整，可作为后续轮次的信任证据复用。</Card>
  <Card title="资本方" icon="chart-bar">结构化数据降低尽调成本。投资记录进入系统，累积投资人信誉与匹配精度。</Card>
  <Card title="服务方" icon="clipboard-list">法务交付被记录并归因。它不再是一次非正式帮忙，而是一份有明确结算依据的正式贡献。</Card>
  <Card title="网络" icon="gem">一个完整闭环进入系统。参与方信誉更新，匹配模型获得新的训练数据。下一次类似交易运行得更高效。</Card>
</CardGrid>

<Callout type="tip">
这就是 WCN 与介绍群、CRM 或积分工具的区别。一次协作不只产出结果；它成为系统资产——每一个闭环都让网络更难被替代。
</Callout>
