## Agent 收入的来源

收入来自四个可独立开票或打包的维度：既可对节点 / 项目直收，也可嵌在服务费与 success fee 里作为**自动化溢价**。

<CardGrid cols={2}>
  <Card title="配置费" icon="settings">为节点或项目部署工作流、工具链、权限模板与合规策略的一次性或项目费；对标实施费（implementation fee）。</Card>
  <Card title="使用费" icon="chart-bar">按席位、按周期、按调用次数或按 token 额度计费；对齐 Snowflake / OpenAI API 类用量模型，便于预测毛利。</Card>
  <Card title="执行费" icon="play">Agent 参与指定里程碑任务（尽调清单、对账、材料生成、多轮评审）且交付物被采纳时计费；与结果弱绑定可降低「只跑不算」纠纷。</Card>
  <Card title="支持费" icon="wrench">监控、调参、红队、人工复核与版本升级；高信任场景下客户愿为 SLA 付 recurring。</Card>
</CardGrid>

## Web3 类比：为什么不是「又一个 Chatbot」

| 协议 / 模式 | 可比点 | 对 WCN 的启示 |
| --- | --- | --- |
| Chainlink 预言机 | 节点为可靠数据交付收费 | Agent 交付的是**可审计工作产物**，应可合同化计费 |
| The Graph 索引者 | 查询与市场激励对齐工作量 | 高频调用 = 更高基础设施成本 = 应对齐用量费 |
| Filecoin 存储合约 | 按约履约、按量付费 | 长周期任务可拆里程碑付费，降低单方风险 |

共同点：**链上或链下，客户为持续可用的服务付对价**；不依赖代币二级市场涨价补贴算力。

## 为什么 Agent 收入成立

<Statement>Agent 的商业价值不来自「模型有多强」，而来自「是否进入真实闭环并被采纳」。</Statement>

在 WCN 中，Agent 承担正式执行角色：接入 Deal Room、服务工单与合规流时，其产出可留痕、可验收，因而具备**独立 SKU** 条件。免费期可用于教育市场，长期必须有价格锚，否则算力与合规成本无法由网络内化。

## 与服务收入、成交收入的关系

- **独立售卖**：Agent Desk、组织级订阅、按部门配额——现金流直接、产品边界清晰。
- **嵌入溢价**：同一笔法务 retainer 或 FA 服务中，含「人机协同交付」的套餐可比纯人工报价更高毛利；success fee 不变时，平台可通过降低交付成本改善 take rate。
- **飞轮**：成交越多 → 模板化工作流越多 → Agent 复用率上升 → 边际成本下降 → 更有空间做价格竞争或提高平台抽成。

<Callout type="tip">
将 Agent 仅定位为「提效工具」会低估其财务属性：在正确计量下，它是**第三类 recurring**（与席位、部分 retainer 并列），也是在熊市中不依赖 Token 补贴的增长杠杆。
</Callout>
