## Agent 放大什么

Agent 承担研究、引入、触达、跟进、监控与协调。它们不闲聊；它们进入限定范围的任务，并留下完整记录。通过扩展结构化执行，它们提高一组固定节点可推进的可验证业务量。

<Statement>Agent 的价值来自进入真实结果并被采纳，而非来自模型有多大。</Statement>

## 治理边界

Agent 在严格边界内运作。这些边界正是让其产出进入记录、而非停留于非正式的条件。

<CardGrid cols={2}>
  <Card title="能力清单" icon="clipboard-list">每个 Agent 有声明的能力边界，固定它可以做什么。</Card>
  <Card title="任务合约" icon="file-text">每项任务携带限定范围的授权，按任务授予，而非常驻。</Card>
  <Card title="PoB 证据" icon="circle-check">每个动作留下 PoB 记录，使工作可追溯、可复核。</Card>
  <Card title="人类背书" icon="hand">Agent 不能签署合约、转移资金或做出对外承诺。须由人类背书。</Card>
</CardGrid>

## Agent 产出如何结算

Agent 工作不按单独的表结算。其产出并入交付物，经 PoB 对账，并在与人类贡献相同的 Carry 瀑布内归因。

<Steps>
  <Step number={1} title="进入限定任务">Agent 依其任务合约，在明确里程碑上行动——尽调、对账或材料生成。</Step>
  <Step number={2} title="留下证据">该动作产出可复核的记录，支持结果主张。</Step>
  <Step number={3} title="验证结果">对账成立后生成验证节点，结果成为已验证。</Step>
  <Step number={4} title="经瀑布归因">已验证的净收益被结算，所贡献的工作在宪章约束的桶下归因。</Step>
</Steps>

## 为什么这对模型重要

Agent 降低结构化执行的边际成本，使网络可在成本不成比例上升的情况下推进更多结果。这一收益通过已验证结果捕获，而非通过 Token 补贴。

<Callout type="tip">
仅把 Agent 视为提效工具会低估其角色。在上述边界内，它是产出已验证结果的执行能力的一部分。它也是一种不依赖 Token 的杠杆。
</Callout>

<Callout type="warning">
Agent 从不以自身权限结算。任何 Agent 未经人类背书不得签署、转移或承诺，每个动作均受对账约束。
</Callout>
