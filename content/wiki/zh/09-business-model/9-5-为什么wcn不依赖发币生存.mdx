## 先把价值压在 Token 上的风险

当业务闭环尚未成立、价值被压在 Token 上时，结构面临叠加的失效。

<CardGrid cols={2}>
  <Card title="叙事脆弱" icon="triangle-alert">无经常性收入时，网络被读作在卖预期，估值依赖故事而非已结算记录。</Card>
  <Card title="合规暴露" icon="triangle-alert">越早把参与当作通往 Token 收益的路径来卖，越靠近多辖区的证券与营销红线。</Card>
  <Card title="节点质量失真" icon="triangle-alert">参与者为价格而来，不为交付而来，节点图谱变成流量池而非协作网。</Card>
  <Card title="抗下行极弱" icon="triangle-alert">当国库与激励以 Token 计价时，下行期同时削减融资、招募与留存。</Card>
</CardGrid>

## 经济模型覆盖运营周期

白皮书在文件内闭合其募资推导。下表为宪章相关输入，而非前瞻承诺。

| 输入 | 数值 |
| --- | --- |
| 理论募资上限 | USD 82–127M |
| 红队现实化中位 | USD 55–60M |
| 团队 Runway | 5–7 年 |
| Token 依赖 | 无 |

<Callout type="info">
USD 55–60M 的红队中位足以支撑 Foundation 与团队五到七年，且不依赖任何 Token 发行。这是经济模型的核心稳健性，而非回报预测。
</Callout>

## 资金用途的优先级

模型为资金用途设定清晰优先级。协议金库也随时间从 5% Carry 桶累积。

<Steps>
  <Step number={1} title="合规与法律地基">Foundation 注册、跨辖区结构与法律顾问优先。</Step>
  <Step number={2} title="团队与 Agent 基础设施">运行网络的核心成本：执行团队与 Agent 层。</Step>
  <Step number={3} title="协议金库">5% Carry 桶持续累积，支撑长期可持续性。</Step>
  <Step number={4} title="生态与节点支持">Deal Flow、服务方池与节点工具。</Step>
</Steps>

## Token：被推迟、被合规门控、被完整分配

Token 存在于设计中，但它是 Phase 3 的可选项（M24 及之后），只在跨辖区合规条件满足后才考虑。它是协调与治理工具，绝非融资生命线，也绝非升值承诺。供给完整分配，无隐藏份额。

| 分配对象 | 占总供给（建议） | 说明 |
| --- | --- | --- |
| 节点预留池（四层合伙人） | 25% | 治理与协调权 |
| 团队与 Foundation | 20% | 多年归属 |
| 协议金库 | 15% | 长期可持续性储备 |
| 生态与激励 | 20% | Deal Flow、服务方、增长 |
| 公众与流动性 | 20% | 合规发行与做市 |

<Callout type="warning">
节点预留池是治理与协调权，不是融资工具，亦不以升值为承诺。它仅在 Phase 3 合规达成后激活，否则作废。节点权益与任何 Token 的法律属性，由发行时外部法律意见确定。比例为建议值，最终经治理与法律流程确定。
</Callout>

## 为什么这反而更稳

顺序是刻意的：先证明网络能自筹资金，再考虑 Token。

<CardGrid cols={2}>
  <Card title="可像 B2B 公司一样核验" icon="search">尽调可检视节点权益、已结算结果与运营周期，而非一套 Token 价格故事。</Card>
  <Card title="可在下行期存活" icon="shield">席位资金及其支撑的运营周期，不随 Token 日线波动，使计划在熊市中保持。</Card>
  <Card title="Token 保持可选" icon="layers">若后续引入，Token 可聚焦协调与长期对齐，而非充当唯一资金来源。</Card>
  <Card title="绑定已结算工作" icon="circle-check">收入在已验证结果处确认，使模型锚定真实业务。</Card>
</CardGrid>

<Statement>先业务，后 Token。先现金流，后长期价值映射。</Statement>
