## 昂贵的一课：先链，后业务

过去五年，「先建链再找场景」是 Web3 最常见的路径，也是失败率最高的路径之一。

<CardGrid cols={2}>
  <Card title="EOS" icon="ban">2018 年融资 41 亿美元，史上最大 ICO，并建成高性能 L1。它缺乏标志性应用，生态始终未成形。2023 年品牌重塑为 Antelope，业务密度仍接近于零。问题不在技术，而在没有业务。</Card>
  <Card title="Internet Computer（ICP）" icon="ban">2021 年上线，Dfinity 基金会融资 1.66 亿美元。技术很强——完整 Web 应用跑在链上——但真实使用稀薄，Token 从 700 美元跌至 5 美元。「它能做什么」和「会有人用吗」是两个不同的问题。</Card>
  <Card title="Aptos 与 Sui" icon="triangle-alert">2022 与 2023 年上线，各自融资超过 3.5 亿美元。吞吐与延迟等技术指标领先，但早期 TVL 依赖流动性激励，激励一停便大幅回落。基础设施先行，业务密度没有跟上。</Card>
  <Card title="L2 浪潮" icon="ban">2023 至 2024 年，超过 40 条新 L2 上线——Blast、Mode、Manta、Mantle 等。多数靠空投吸引 TVL，激励停止后真实用户与交易量极低。先建桥、再找货：这一模式反复失败。</Card>
</CardGrid>

<Callout type="warning">
这些项目有一个共同点，且不是技术不好：它们在没有业务密度时引入了链上复杂度。身份、资产、治理、监管、用户体验的成本全部前置，却没有足够的真实交易来承担。
</Callout>

## 行得通的路：先业务，后链

反过来看，Web3 最成功的链上项目几乎都遵循相反的次序——先做业务，再建链。

<CardGrid cols={2}>
  <Card title="Binance → BNB Chain" icon="circle-check">Binance 2017 年先做中心化交易所，积累数千万用户与深厚的交易数据。BNB Chain 2020 年跟上，借既有流量迅速成为 DeFi 中心。链的价值建立在业务之上。</Card>
  <Card title="Coinbase → Base" icon="circle-check">Coinbase 用十年建成大型合规交易所，拥有约 1.1 亿认证用户。Base L2 2023 年上线，第一天即承载真实交易量，无需空投刺激。Base 的成功靠 Coinbase 的业务密度，而非仅靠 OP Stack。</Card>
  <Card title="Stripe → 加密支付" icon="circle-check">Stripe 用十四年建成大规模在线支付，处理数万亿美元。它 2024 年收购稳定币支付公司 Bridge，凭借既有的商户网络与合规体系。业务密度决定了加密层的落地节奏。</Card>
  <Card title="Circle 与 USDC" icon="circle-check">Circle 不先建链，而先做稳定币发行与合规框架。USDC 在多条链上流通，Circle 的价值绑定在它的业务关系上——银行、交易所、商户——而非任何一条链。</Card>
</CardGrid>

> 规律：对成功上链的项目，链不是起点。它是业务密度越过临界点后的自然延伸。先备好货物，再修路——而非反过来。

## WCN 为什么必须先做业务

如果 WCN 今天就强推完整上链，会撞上四个问题。

<CardGrid cols={2}>
  <Card title="节点身份上链" icon="link">节点尚无足够的协作记录证明身份的价值。链上 NFT 变成收藏品，而非工作凭证。</Card>
  <Card title="PoB 记录上链" icon="mail">闭环的数量与质量尚不足以支撑一个可信的价值账本。链上数据读起来是噪音，而非信号。</Card>
  <Card title="结算层上链" icon="scale">结算的对象与规则仍在迭代。过早固化到智能合约里，会束缚业务模型。</Card>
  <Card title="治理上链" icon="landmark">节点与社群的成熟度尚不足以承担去中心化治理的协调成本。链上投票变成少数人的游戏。</Card>
</CardGrid>

<Callout type="info">
核心张力：上链的好处——抗篡改、透明、去信任——只有在业务密度足够时才会显现。低于这个门槛，上链只会增加复杂度、拖慢迭代、抬高冷启动门槛。
</Callout>

## 分阶段路径

<CardGrid cols={1}>
  <Card title="阶段一：Web2 工具与节点网络" icon="globe">WCN 用成熟的 Web2 基础设施——Next.js 应用、数据库、API——招募节点、跑通 Deal 与 Task、启动 Agent 执行。验证的是协作模型，而非链上架构。</Card>
  <Card title="阶段二：PoB 账本积累密度" icon="clipboard-list">闭环持续发生，证据与归因规则在实践中稳定下来。当 PoB 账本拥有足够的密度与可信度时，它才值得被永久记录。</Card>
  <Card title="阶段三：选择性链上承载" icon="link-2">WCN 把最需要抗篡改的数据——PoB 摘要、节点信誉快照、结算记录——迁移上链。它在成本、速度与合规之间选择合适的链，而非绑定任何一条。</Card>
  <Card title="阶段四：资产层与治理层" icon="building-2">待网络成熟、PoB 账本可信、结算规则稳定后，WCN 再考虑链上资产发行与去中心化治理。届时，上链是水到渠成，而非强行推进。</Card>
</CardGrid>

## 常见质疑

<CardGrid cols={2}>
  <Card title="这不就是 Web2 吗？" icon="circle-help">不是。终局包含链上身份、链上 PoB、链上结算以及可能的 Token。这些层只有在业务网络成熟后才有意义。Web2 工具是手段，不是终点。</Card>
  <Card title="不上链怎么证明可信？" icon="circle-help">短期内，可信来自业务结果本身——Deal 是否成交、项目是否拿到融资、服务是否交付。上链是长期的信任层，而非当下唯一的信任来源。</Card>
  <Card title="竞争对手会先上链抢市场吗？" icon="circle-help">历史表明：没有业务密度的链上项目，即便先发也形不成护城河。真正的优势——节点密度、PoB 数据、Agent 训练质量——需要时间，无法靠上链买到。</Card>
  <Card title="投资人会嫌不够 Web3 吗？" icon="circle-help">审慎的投资人看业务逻辑是否成立，而非技术标签。Binance 早期是纯中心化产品，它的业务密度为 BNB Chain 提供了真实基础。WCN 遵循同一逻辑。</Card>
</CardGrid>

<Callout type="tip">
WCN 的终局可以有链、有 Token、有完整的链上治理。它的起点必须是业务网络。链是结果，不是前提——这条路径已被 Binance、Coinbase、Stripe 反复验证。
</Callout>
