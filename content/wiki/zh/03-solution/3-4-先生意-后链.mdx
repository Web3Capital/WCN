## 行业最昂贵的教训：先链后业务

过去五年，「先建链再找场景」是 Web3 最常见的创业路径，也是失败率最高的路径之一。

<CardGrid cols={2}>
  <Card title="EOS" icon="ban">2018 年融资 $4.1B（史上最大 ICO），构建了高性能 L1。但缺乏杀手级应用，生态始终未起。2023 年品牌重塑为 Antelope，但业务密度仍接近于零。技术不是问题，没有业务才是。</Card>
  <Card title="ICP (Internet Computer)" icon="ban">2021 年上线，Dfinity 基金会融资 $166M。技术能力极强（链上跑 Web 应用），但缺乏真实使用场景。Token 从 $700 跌至 $5。「能做什么」和「有人用吗」是两个完全不同的问题。</Card>
  <Card title="Aptos / Sui" icon="triangle-alert">2022-2023 年上线，分别融资 $350M+。技术指标（TPS、延迟）领先，但早期 TVL 高度依赖流动性激励。激励结束后 TVL 大幅回落。基础设施先行，但业务密度不足。</Card>
  <Card title="无数 L2" icon="ban">2023-2024 年超过 40 条新 L2 上线（Blast, Mode, Manta, Mantle...）。大多数通过空投预期吸引 TVL，但去除激励后真实用户和交易量极低。「先建桥，再找货」的模式反复失败。</Card>
</CardGrid>

<Callout type="warning">
这些项目的共同点不是技术不好，而是**在没有业务密度的情况下引入了链上复杂度**。身份、资产、治理、监管、用户体验的成本全部前置，但系统里没有足够的真实交易来支撑这些成本。
</Callout>

## 成功路径：先业务，后链

反过来看，Web3 领域最成功的链上项目，几乎都遵循「先有业务，后建链」的路径：

<CardGrid cols={2}>
  <Card title="Binance → BNB Chain" icon="circle-check">Binance 2017 年先做中心化交易所（业务），积累了数千万用户和海量交易数据。2020 年推出 BNB Chain（链），因为已有 Binance 的用户导流和流量支撑，BNB Chain 快速成为 DeFi 主阵地。链的价值建立在业务之上。</Card>
  <Card title="Coinbase → Base" icon="circle-check">Coinbase 用十年时间建成全球最大合规交易所（业务），拥有 1.1 亿认证用户。2023 年推出 Base L2（链），上线即有真实交易量，无需空投刺激。Base 的成功不是因为 OP Stack 技术好，而是因为 Coinbase 的业务密度足够。</Card>
  <Card title="Stripe → 加密支付" icon="circle-check">Stripe 先用 14 年建成全球最大在线支付基础设施（业务），处理数万亿美元交易。2024 年收购 Bridge（稳定币支付），因为已有商户网络和合规体系。业务密度决定了加密层的落地速度。</Card>
  <Card title="Circle / USDC" icon="circle-check">Circle 不是先做公链，而是先做稳定币发行和合规框架。USDC 在多条链上流通，Circle 的价值不绑定任何一条链，而是绑定它的业务关系网络（银行、交易所、商户）。</Card>
</CardGrid>

> **规律**：成功的链上项目，链不是起点，是业务密度到达临界点后的自然延伸。先有货物，再修公路——而不是先修公路，等货物来。

## WCN 为什么必须先做业务

如果 WCN 今天就全力推链上化，会遇到什么：

<CardGrid cols={2}>
  <Card title="节点身份上链" icon="link">但节点还没有足够的协作记录来证明身份的价值。链上 NFT 变成收藏品，而非工作凭证。</Card>
  <Card title="PoB 记录上链" icon="mail">但闭环数量和质量还不足以支撑一个可信的价值账本。链上数据变成噪音，而非信号。</Card>
  <Card title="结算层上链" icon="scale">但结算对象和规则还在迭代。过早固化到智能合约里，反而限制了业务模型的灵活性。</Card>
  <Card title="治理上链" icon="landmark">但社区和节点的成熟度还不足以承受去中心化治理的协调成本。链上投票变成少数人的游戏。</Card>
</CardGrid>

<Callout type="info">
核心矛盾：链上化的好处（不可篡改、透明、去信任）只有在业务密度足够时才能发挥。在业务密度不足时，链上化只增加复杂度、降低迭代速度、抬高冷启动门槛。
</Callout>

## WCN 的分阶段路径

<CardGrid cols={1}>
  <Card title="阶段一：Web2 工具 + 节点网络" icon="globe">用成熟的 Web2 基础设施（Next.js 应用、数据库、API）先把节点招入系统、让 Deal 和 Task 跑起来、让 Agent 开始执行。验证的是协作模型，而非链上架构。</Card>
  <Card title="阶段二：PoB 账本积累密度" icon="clipboard-list">让闭环持续发生，让证据和归因规则在实践中稳定下来。当 PoB 账本有足够密度和可信度时，它才值得被永久记录。</Card>
  <Card title="阶段三：选择性链上承载" icon="link-2">把最需要不可篡改性的数据（PoB 摘要、节点信誉快照、结算记录）逐步迁移到链上。选择最合适的链（成本、速度、合规性），而非被任何一条链绑定。</Card>
  <Card title="阶段四：资产层和治理层" icon="building-2">当业务网络成熟、PoB 账本可信、结算规则稳定后，再考虑链上资产发行和去中心化治理。这时链上化是水到渠成，而非强行推进。</Card>
</CardGrid>

## 常见质疑与回应

<CardGrid cols={2}>
  <Card title="「这不就是 Web2 吗？」" icon="circle-help">不是。WCN 的终局包含链上身份、链上 PoB、链上结算和可能的 Token 激励。但这些层只有在业务网络成熟后才有意义。Web2 工具是手段，不是终点。</Card>
  <Card title="「不上链怎么证明可信？」" icon="circle-help">短期内，可信度来自业务结果本身——Deal 是否真的成交了、项目是否真的拿到融资了、服务是否真的交付了。链上化是长期的信任增强层，但不是当下唯一的信任来源。</Card>
  <Card title="「竞争对手会先上链抢市场」" icon="circle-help">历史反复证明：没有业务密度的链上项目，即使先发也无法形成护城河。真正的竞争优势是节点网络密度、PoB 数据积累和 Agent 训练质量——这些都需要时间，且无法通过单纯上链获得。</Card>
  <Card title="「投资人会不会觉得不够 Web3？」" icon="circle-help">聪明的投资人看的是业务逻辑是否成立，而非技术标签。Binance 早期也是纯中心化产品，但它的业务密度为后来的 BNB Chain 提供了真实基础。WCN 的逻辑完全一致。</Card>
</CardGrid>

<Callout type="tip">
WCN 的终局可以有链、有 Token、有完整的链上治理。但它的起点必须是业务网络。链是结果，不是前提——这是被 Binance、Coinbase、Stripe 反复验证的路径。
</Callout>
