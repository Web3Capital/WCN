# Node NFT 把权益绑定到责任，而非绑定到回报承诺。
</Hero>

<MetaGrid>
  <Meta label="定义" value="承载身份、权限与历史指针的链上凭证" />
  <Meta label="它不是什么" value="承诺固定回报、开放投机、与交付脱钩的凭证" />
  <Meta label="标准" value="标准节点采用 ERC-721 加 ERC-6551（Token Bound Account）" />
</MetaGrid>

## 合伙人 NFT 是一份合伙人合约

对四个合伙人层——创始、国家、城市、赛道——NFT 在一份凭证中把权益绑定到责任。权益覆盖治理、分润与 Deal Flow 优先级。责任覆盖四项：资源引入、业务推进、证据提交、合规遵从。

ERC-6551 Token Bound Account 让每个节点拥有自己的账户与履约记录。节点承载自己的历史，而非散落在多个钱包中。

## Node NFT 与常见身份凭证的对照

| 凭证 | 强项 | 局限 | WCN 的取舍 |
| --- | --- | --- | --- |
| ENS | 人类可读、广泛集成 | 不表达复杂权限或组织角色 | 与 ENS 并列：ENS 指人或品牌，Node NFT 指机构席位 |
| 灵魂绑定代币 | 不可转让、声誉绑定 | 升级、席位迁移或合规冻结场景不灵活 | 采用可转移性由政策决定的凭证——可锁定、映射可收回 |
| W3C 可验证凭证 | 标准隐私模型、选择性披露 | 链上可组合性与钱包展示较弱 | 关键声明双写：VC 用于合规披露，NFT 用于协议内快速门禁 |

## Node NFT 承载什么

<CardGrid cols={2}>
  <Card title="身份" icon="id-card">在链上证明某地址被绑定到特定节点与席位。权限刷新通过 metadata 版本，而非更换钱包。</Card>
  <Card title="权限" icon="key">它是触达 API、合约函数或路由的必要非充分条件。服务端访问控制与合规标记仍然管辖。</Card>
  <Card title="记录" icon="chart-bar">它可承载信誉分、等级与任务历史指针。完整证据留在链下存储，以避免 gas 与隐私成本。</Card>
  <Card title="可编程性" icon="link-2">它支持激励合约、联合白名单与跨合作伙伴集成。合约层须防止任何被表述为证券或存款的可能。</Card>
</CardGrid>

## Node NFT 不是什么

<CardGrid cols={2}>
  <Card title="不是理财产品" icon="ban">它不承诺本金、固定回报或自动分润。它与基金或存款的法律边界相隔离。</Card>
  <Card title="不是头像" icon="ban">视觉层是次要的。价值来自与 WCN 账户、协议与责任的绑定。</Card>
  <Card title="不是开放铸造" icon="ban">铸造权属于治理与运营。开放铸造会摧毁席位稀缺性与风控基础。</Card>
  <Card title="不是万能钥匙" icon="ban">私钥丢失、合约终止或合规触发，通过冻结映射、轮换地址或作废凭证处置。</Card>
</CardGrid>

## 先有供给，再有凭证

<Callout type="warning">节点是有责任的席位，不是花钱买的身份。网络先建立真实的 Deal、任务、Reviewer 与证据标准。然后 NFT 才承载这一结构。颠倒顺序——在责任存在之前出售凭证——是空壳发行。</Callout>

<Statement>Node NFT 是专业网络的凭证：链上可验证、链下可问责、系统可收回。它不是投机标的。</Statement>
