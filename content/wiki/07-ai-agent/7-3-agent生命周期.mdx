## Agent 的关键状态

<CardGrid cols={3}>
  <Card title="Registered" description="Agent 已被登记，但还没进入正式业务流程。" />
  <Card title="Authorized" description="Agent 被赋予明确任务、数据和权限边界。" />
  <Card title="Active" description="Agent 正在执行任务并持续写入日志。" />
  <Card title="Paused / Suspended" description="因质量、越权或风险问题暂停。" />
  <Card title="Upgraded" description="Agent 升级模型能力或权限等级。" />
  <Card title="Retired" description="Agent 因不再适用或不达标而退出系统。" />
</CardGrid>

## 生命周期路径

<Callout type="warning">
注册 → 授权 → 上线 → 日志记录 → 审核 / 复核 → 升级 / 暂停 / 淘汰
</Callout>

Agent 生命周期的设计目标，不是增加复杂度，而是保证：

<CardGrid cols={2}>
  <Card title="每个 Agent 都有身份" description="知道是谁、做什么、属于谁。" />
  <Card title="每个 Agent 都有边界" description="不能无约束读取和执行。" />
  <Card title="每个 Agent 都有日志" description="可以回溯与审查。" />
  <Card title="每个 Agent 都能被淘汰" description="质量差或越权的 Agent 不能无限存在。" />
</CardGrid>

## 为什么这一页重要？

因为如果 Agent 没有生命周期，系统会出现两个问题：

<CardGrid cols={2}>
  <Card title="Agent 失控" description="没有授权与暂停机制，Agent 容易越权或持续输出低质量结果。" />
  <Card title="Agent 无法进入 Proof" description="没有身份和日志，Agent 的贡献无法被验证和结算。" />
</CardGrid>
