## 五种典型 Agent

<CardGrid cols={2}>
  <Card title="Research Agent" description="负责信息收集、摘要、初筛、对比、尽调辅助与结构化输出。" />
  <Card title="Deal Agent" description="负责项目与资本匹配、引荐准备、状态推进与 deal 推荐。" />
  <Card title="Growth Agent" description="负责内容生成、传播辅助、增长归因与渠道支持。" />
  <Card title="Execution Agent" description="负责纪要、待办、提醒、回执收集、节点协作节奏管理。" />
  <Card title="Liquidity Agent" description="负责流动性监控、上所准备、市场状态与基础策略支持。" />
</CardGrid>

## 它们最本质的区别是什么？

<CardGrid cols={2}>
  <Card title="Research Agent 面向信息" description="核心是理解和整理。" />
  <Card title="Deal Agent 面向匹配" description="核心是连接与推进。" />
  <Card title="Growth Agent 面向传播" description="核心是内容与增长动作。" />
  <Card title="Execution Agent 面向流程" description="核心是确保任务被完成。" />
  <Card title="Liquidity Agent 面向市场" description="核心是交易与流动性状态。" />
</CardGrid>

## 为什么一定要把 Agent 分类？

<Callout type="warning">
因为不分类，权限会混乱，日志会混乱，责任会混乱，价值也无法被准确结算。
</Callout>

分类清楚后，系统才能知道：

<CardGrid cols={2}>
  <Card title="谁能看什么" description="不同 Agent 的可见数据不同。" />
  <Card title="谁能做什么" description="不同 Agent 的执行动作不同。" />
  <Card title="谁该留下什么日志" description="不同任务类型对应不同留痕标准。" />
  <Card title="谁如何进入 PoB" description="只有被采纳并进入闭环的 Agent 输出才有价值。" />
</CardGrid>
