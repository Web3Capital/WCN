## Agent 怎么进入闭环？

Agent 不是天然属于业务闭环，它必须通过正式工作流进入。

<Callout type="warning">
任务派发 → Agent 执行 → 输出被记录 → 人类节点采纳 → 输出进入结果链 → Proof Desk 审核 → PoB 记账
</Callout>

只有这样，Agent 的价值才不是抽象"有帮助"，而是可被系统识别的结果推动力。

## Agent 的输出怎样进入 Proof？

没有这些条件，Agent 的输出就只能算"工具生成物"，不能进入系统的长期价值层。

<CardGrid cols={2}>
  <Card title="任务 ID 绑定" description="输出必须属于某个正式任务，而不是无主结果。" />
  <Card title="输入与版本留痕" description="必须知道 Agent 依据什么输入得出什么输出。" />
  <Card title="人类采纳记录" description="需要有节点确认这个输出被使用了，并影响了真实结果。" />
  <Card title="结果关联" description="输出要能被追溯到某个 deal、task 或闭环事件。" />
</CardGrid>

## 为什么这一页重要？

<CardGrid cols={2}>
  <Card title="它决定 AI 是否只是工具" description="只有进入闭环，AI 才会成为正式执行层。" />
  <Card title="它决定 Agent 是否可被结算" description="没有 Proof 就没有 PoB，也就没有长期价值。" />
  <Card title="它决定 Agent 是否可治理" description="只有进入正式系统才可能被审核、暂停、升级或淘汰。" />
  <Card title="它决定 WCN 的差异化" description="这一步正是 WCN 和普通 AI 工具平台最大的区别。" />
</CardGrid>

<Callout type="warning">
WCN 的关键不是"用了 AI"，而是"让 AI 第一次进入了可验证、可归因、可结算的业务网络"。
</Callout>
