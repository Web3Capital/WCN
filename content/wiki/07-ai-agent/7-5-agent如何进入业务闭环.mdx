## Agent 进入闭环的路径

Agent 不是天然属于业务闭环，它必须通过正式工作流进入。

<Callout type="info">
任务派发 → Agent 执行 → 输出被记录 → 人类节点采纳 → 输出进入结果链 → Proof Desk 审核 → PoB 记账
</Callout>

<Statement>Agent 的价值不是抽象的「有帮助」，而是可被系统识别的结果推动力。</Statement>

---

## Agent 输出进入 Proof 的条件

<CardGrid cols={2}>
  <Card title="任务 ID 绑定" icon="🔗">输出必须属于某个正式任务，而不是无主结果。</Card>
  <Card title="输入与版本留痕" icon="📋">必须知道 Agent 依据什么输入得出什么输出。</Card>
  <Card title="人类采纳记录" icon="✅">需要有节点确认这个输出被使用了，并影响了真实结果。</Card>
  <Card title="结果关联" icon="🔄">输出要能被追溯到某个 deal、task 或闭环事件。</Card>
</CardGrid>

<Callout type="warning">
没有这些条件，Agent 的输出只能算「工具生成物」，不能进入系统的长期价值层。
</Callout>

---

## 这一步决定了什么

<CardGrid cols={2}>
  <Card title="AI 是否只是工具" icon="🤖">只有进入闭环，AI 才会成为正式执行层。</Card>
  <Card title="Agent 是否可被结算" icon="💰">没有 Proof 就没有 PoB，也就没有长期价值。</Card>
  <Card title="Agent 是否可治理" icon="🛡️">只有进入正式系统才可能被审核、暂停、升级或淘汰。</Card>
  <Card title="WCN 的差异化" icon="⚡">这一步正是 WCN 和普通 AI 工具平台最大的区别。</Card>
</CardGrid>

<Callout type="tip">
WCN 的关键不是「用了 AI」，而是「让 AI 第一次进入了可验证、可归因、可结算的业务网络」。
</Callout>
