## Agent 收入的来源

Agent 收入来自配置、使用、执行和持续支持四个维度。既可以是对节点和项目的直接收入，也可以是嵌入服务和 Deal 中的间接收入。

<CardGrid cols={2}>
  <Card title="配置费" icon="⚙️">为节点或项目配置特定 Agent 体系。</Card>
  <Card title="使用费" icon="📊">按周期、权限或额度使用 Agent。</Card>
  <Card title="执行费" icon="▶️">Agent 参与特定任务和闭环时产生的执行收入。</Card>
  <Card title="支持费" icon="🔧">围绕 Agent 的运维、调整、优化和审核支持。</Card>
</CardGrid>

---

## 为什么 Agent 收入成立

<Statement>Agent 的商业价值，不来自「模型有多强」，而来自「它是否进入了真实闭环」。</Statement>

Agent 不再只是「帮你写点东西」，而是在 WCN 中承担正式执行角色。只要它能进入任务、产出结果并被采纳，它就具备商业化基础。

---

## Agent 收入和服务收入的关系

有些 Agent 收入可以独立存在，例如 Agent Desk 或 Agent Subscription。也有一些 Agent 收入会嵌入到服务包和 Deal 收入中，成为隐藏在服务层中的自动化能力溢价。

<Callout type="tip">
Agent 收入的独立性和嵌入性并存——既能单独售卖，也能作为服务溢价。
</Callout>
