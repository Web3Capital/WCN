## 当前 Web3 服务的结构性问题

<CardGrid cols={2}>
  <Card title="碎片化" icon="smartphone">法务找一家、审计找一家、做市找一家、增长找一家。每家都是独立沟通，没有统一的协调界面。一个项目方可能同时管理 8 个 Telegram 群和 5 个 Google Doc。</Card>
  <Card title="无标准交付" icon="clipboard-list">服务商说「做完了」，但没有标准化的交付确认机制。审计报告的质量参差不齐，法律意见的覆盖范围不明确，增长服务的效果没有归因。</Card>
  <Card title="无结算归因" icon="banknote">服务费通常是预付或里程碑付款，与最终业务结果无关。做市商拿了费用但流动性没改善，增长团队拿了费用但用户没增长——没有系统把服务和结果绑定。</Card>
  <Card title="无规模化路径" icon="ruler">优秀的 FA、律师、审计师的时间是有限的。他们的方法论无法被复制，导致服务能力受限于个人产能。</Card>
</CardGrid>

## WCN 的执行与服务层如何运作

<Steps>
  <Step title="服务注册">服务节点录入能力：法务（合同/结构/合规）、审计（智能合约/财务）、安全、品牌、增长、开发、研究等。能力进入 Service Registry，可被 Deal 调用。</Step>
  <Step title="Task 分配">当 Deal 需要服务时，系统从 Service Registry 匹配合适的服务节点。每个 Task 有明确的 owner、scope、deadline 和验收标准。</Step>
  <Step title="执行推进">人类服务节点和 Agent 协同推进。Agent 负责信息整理、催办、纪要和状态追踪，人类负责判断、交付和签字。</Step>
  <Step title="交付与证据">服务完成后，交付件进入 Evidence Packet——法律意见书、审计报告、交付 memo、SOW 确认等。交付质量由 Reviewer 审核。</Step>
</Steps>

## 这一层的参与方

<CardGrid cols={2}>
  <Card title="法务 / 合规" icon="scale">SAFE/SAFT 结构审核、Token 法律意见、实体设立、监管咨询、投资人 KYC/AML。在 Web3 中，法务是最影响 Deal 速度的环节——一份法律意见可能卡住整个融资 2 周。</Card>
  <Card title="审计 / 安全" icon="lock">智能合约审计、财务审计、安全评估、渗透测试。高质量审计是项目获得投资和上所的前置条件。</Card>
  <Card title="开发 / 研究" icon="laptop">技术开发、架构评审、研究报告、竞品分析、Token 经济设计。Research Agent 可以覆盖 80% 的信息整理工作。</Card>
  <Card title="增长 / BD" icon="rocket">品牌策略、内容创作、社区建设、KOL 合作、用户获取。Growth Agent 可以标准化分发流程和归因追踪。</Card>
</CardGrid>

## Agent 在执行层的角色

<CardGrid cols={2}>
  <Card title="Research Agent" icon="microscope">项目摘要、竞品分析、风险标记、市场数据整理。把分析师 3 天的工作压缩到 30 分钟，人类审核后采纳。</Card>
  <Card title="Execution Agent" icon="zap">纪要自动生成、待办追踪、催办提醒、材料收集、状态更新。消除 Deal 中 30-40% 的协调时间。</Card>
</CardGrid>

<Statement>Agent 不替代服务节点的专业判断，而是放大他们的产能——让一个优秀律师的方法论服务 10 个 Deal 而不是 3 个。</Statement>

<Callout type="tip">
执行与服务层位于资本之后、分发之前，是最直接决定「Deal 能不能做成」的关键桥梁。没有这一层，WCN 只是一个高级版的介绍平台。
</Callout>
